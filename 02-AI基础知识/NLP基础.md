# NLP基础

> 理解自然语言处理的核心概念、任务流程与技术演进，为大语言模型学习奠定基础

---

## 一、NLP概述

### 1.1 什么是NLP

**自然语言处理（Natural Language Processing，NLP）** 是人工智能的一个分支，研究如何让计算机理解、处理和生成人类语言。NLP位于计算机科学、语言学和人工智能的交叉领域。

```
NLP核心能力：

人类语言 ──→ [NLP系统] ──→ 机器理解/生成
    │                           │
    ▼                           ▼
  文本/语音                  语义表示/回复
```

### 1.2 NLP的层次结构

```
NLP处理层次（从低到高）：

┌─────────────────────────────────────┐
│  语用层（Pragmatics）                │  ← 语境、意图、言外之意
├─────────────────────────────────────┤
│  语义层（Semantics）                 │  ← 词义、句义、逻辑关系
├─────────────────────────────────────┤
│  句法层（Syntax）                    │  ← 句子结构、语法规则
├─────────────────────────────────────┤
│  词法层（Morphology）                │  ← 词性、词形变化
├─────────────────────────────────────┤
│  语音层/字形层（Phonology/Orthography）│  ← 发音、文字形态
└─────────────────────────────────────┘
```

### 1.3 NLP vs NLU vs NLG

| 术语 | 全称 | 含义 | 示例 |
|------|------|------|------|
| **NLP** | Natural Language Processing | 自然语言处理（总称） | 包含理解和生成 |
| **NLU** | Natural Language Understanding | 自然语言理解 | 情感分析、意图识别 |
| **NLG** | Natural Language Generation | 自然语言生成 | 文本摘要、机器翻译 |

```
NLP = NLU + NLG

用户输入 ──→ [NLU] ──→ 语义理解 ──→ [NLG] ──→ 生成回复
              │                          │
              ▼                          ▼
          意图识别                      文本生成
          实体提取                      摘要翻译
```

## 二、NLP核心任务

### 2.1 文本预处理

文本预处理是NLP pipeline的第一步，将原始文本转换为适合模型处理的形式。

```
文本预处理流程：

原始文本
    │
    ▼
┌─────────────┐
│  文本清洗    │  ← 去除HTML标签、特殊字符、多余空格
└─────────────┘
    │
    ▼
┌─────────────┐
│  分词（Tokenization）│  ← 将句子切分为词或子词单元
└─────────────┘
    │
    ▼
┌─────────────┐
│  标准化      │  ← 大小写转换、词形还原、去停用词
└─────────────┘
    │
    ▼
┌─────────────┐
│  向量化      │  ← 将文本转换为数值向量
└─────────────┘
    │
    ▼
模型输入
```

#### 分词（Tokenization）

分词是将连续的文本切分为离散单元（token）的过程。

| 语言 | 分词特点 | 示例 |
|------|---------|------|
| **英文** | 按空格分词，处理时态/单复数 | "running" → "run" + "ing" |
| **中文** | 无空格，需专门分词算法 | "自然语言处理" → "自然/语言/处理" |
| **日文** | 混合汉字、平假名、片假名 | 需要多语言分词器 |

**常见分词方法**：

| 方法 | 原理 | 代表工具 |
|------|------|---------|
| **基于规则** | 词典匹配、最大匹配法 | 结巴分词（Jieba） |
| **基于统计** | HMM、CRF序列标注 | HanLP、THULAC |
| **基于深度学习** | 神经网络预测切分点 | BERT Tokenizer |
| **子词分词** | BPE、WordPiece、SentencePiece | GPT、BERT使用 |

#### 子词分词（Subword Tokenization）

```
BPE（Byte Pair Encoding）示例：

初始词表：{a, b, c, d, e, ...}

训练语料：
- low: l, o, w
- lower: l, o, w, e, r
- newest: n, e, w, e, s, t
- widest: w, i, d, e, s, t

高频合并：
1. es 出现最多 → 加入词表
2. est 出现最多 → 加入词表
3. lo 出现最多 → 加入词表
4. low 出现最多 → 加入词表

最终词表包含：{a, b, c, ..., es, est, lo, low, ...}

优势：平衡词表大小和表达能力，处理未登录词
```

### 2.2 词法分析

#### 词性标注（POS Tagging）

为句子中的每个词标注词性（名词、动词、形容词等）。

```
词性标注示例：

自然/ADJ 语言/NOUN 处理/NOUN 是/VERB 人工智能/NOUN 的/PART 核心/ADJ 技术/NOUN 。/PUNCT
```

#### 命名实体识别（NER）

识别文本中具有特定意义的实体（人名、地名、组织名等）。

```
NER示例：

[OpenAI]/ORG 于 [2022年11月]/TIME 发布了 [ChatGPT]/PRODUCT 。

实体类型：
- ORG：组织
- PER：人名
- LOC：地点
- TIME：时间
- PRODUCT：产品
```

### 2.3 句法分析

#### 依存句法分析

分析句子中词与词之间的依存关系。

```
依存句法分析示例：

        处理（ROOT）
       /    \
    语言      是
    /         \
  自然        技术
             /   \
           核心   的
                  |
               人工智能

关系：
- 语言 ←[定语]- 自然
- 处理 ←[主语]- 语言
- 处理 ←[谓语]- 是
- 技术 ←[表语]- 是
- 技术 ←[定语]- 核心
- 技术 ←[定语]- 人工智能
```

### 2.4 语义分析

#### 词义消歧（WSD）

确定多义词在特定上下文中的含义。

```
词义消歧示例：

"苹果"在不同语境中的含义：
- "我买了一斤苹果" → 水果
- "苹果公司发布了新iPhone" → 科技公司
- "牛顿被苹果砸中" → 水果（历史典故）
```

#### 语义角色标注（SRL）

识别句子中各成分扮演的语义角色。

```
SRL示例：

"小明昨天在图书馆借了一本书"

- 施事（Agent）：小明
- 时间（Time）：昨天
- 地点（Location）：图书馆
- 动作（Predicate）：借
- 受事（Theme）：一本书
```

## 三、NLP主要应用任务

### 3.1 文本分类

将文本分配到预定义的类别中。

| 应用 | 说明 | 示例 |
|------|------|------|
| **情感分析** | 判断文本情感倾向 | 评论是正面/负面/中性 |
| **主题分类** | 识别文本主题 | 新闻属于体育/财经/科技 |
| **垃圾检测** | 识别垃圾信息 | 垃圾邮件、垃圾评论 |
| **意图识别** | 识别用户意图 | "订机票"→预订意图 |

```
情感分析示例：

输入："这款手机的拍照效果太棒了，夜景模式非常清晰！"
      │
      ▼
  [情感分析模型]
      │
      ▼
  输出：正面（置信度：0.95）
```

### 3.2 序列标注

为序列中的每个元素标注标签。

| 任务 | 说明 | 示例 |
|------|------|------|
| **命名实体识别** | 标注实体边界和类型 | "[北京]/LOC 的 [张三]/PER" |
| **词性标注** | 标注每个词的词性 | "自然/ADJ 语言/NOUN" |
| **分词** | 标注词边界 | "自然/语言/处理" |

### 3.3 文本生成

根据输入生成自然语言文本。

| 任务 | 说明 | 示例 |
|------|------|------|
| **机器翻译** | 将一种语言翻译为另一种 | "Hello" → "你好" |
| **文本摘要** | 生成文本的简短摘要 | 长文 → 一句话摘要 |
| **对话生成** | 生成对话回复 | 用户问 → AI答 |
| **文本续写** | 根据前文续写 | "从前有座山" → "山里有座庙" |

### 3.4 信息抽取

从非结构化文本中提取结构化信息。

| 任务 | 说明 | 示例 |
|------|------|------|
| **实体抽取** | 提取命名实体 | 从简历中提取姓名、学校 |
| **关系抽取** | 提取实体间关系 | "张三-就职于-阿里巴巴" |
| **事件抽取** | 提取事件要素 | 时间、地点、参与者 |

### 3.5 问答系统

根据问题从文本中找出答案或生成答案。

| 类型 | 说明 | 示例 |
|------|------|------|
| **抽取式QA** | 从原文中抽取答案片段 | 阅读理解 |
| **生成式QA** | 生成自然语言答案 | ChatGPT对话 |
| **知识库QA** | 基于结构化知识回答 | "周杰伦的生日是？" |

## 四、NLP技术演进

### 4.1 基于规则的方法（1950s-1990s）

```
基于规则的NLP：

输入文本
    │
    ▼
┌─────────────┐
│  规则库      │  ← 人工编写的语法规则、词典
│  IF-THEN    │
└─────────────┘
    │
    ▼
  输出结果

特点：
- 可解释性强
- 维护成本高
- 泛化能力差
- 需要领域专家
```

### 4.2 基于统计的方法（1990s-2010s）

```
基于统计的NLP：

输入文本
    │
    ▼
┌─────────────┐
│  特征提取    │  ← 人工设计特征模板
└─────────────┘
    │
    ▼
┌─────────────┐
│  统计模型    │  ← HMM、CRF、SVM
└─────────────┘
    │
    ▼
  输出结果

特点：
- 数据驱动
- 需要特征工程
- 效果依赖特征质量
```

### 4.3 基于神经网络的方法（2010s-2017）

```
神经网络NLP：

输入文本 → 词向量（Word2Vec/GloVe） → 神经网络（RNN/CNN） → 输出

代表技术：
- Word2Vec（2013）：词嵌入学习
- Seq2Seq（2014）：编码器-解码器架构
- Attention（2015）：注意力机制
```

### 4.4 预训练+微调范式（2018-2022）

```
预训练 + 微调：

大规模无标注语料
        │
        ▼
   [预训练模型]  ← 学习通用语言表示
        │
        ▼
    通用模型（BERT/GPT）
        │
        ├──→ 下游任务1：文本分类（微调）
        ├──→ 下游任务2：NER（微调）
        └──→ 下游任务3：问答（微调）

优势：
- 预训练一次，多任务复用
- 小样本即可微调
- 效果远超从头训练
```

### 4.5 大模型时代（2022至今）

```
大模型范式：

海量数据 + 超大模型 → 预训练 → 通用大模型
                              │
                              ├──→ Prompt工程（零样本/少样本）
                              ├──→ 指令微调（Instruction Tuning）
                              └──→ 人类反馈强化学习（RLHF）

特点：
- 涌现能力
- 上下文学习
- 通用任务处理
- 提示工程替代微调
```

## 五、NLP核心概念详解

### 5.1 词嵌入（Word Embedding）

将离散的词映射到连续的向量空间，使语义相似的词在向量空间中距离相近。

```
词嵌入空间可视化：

        国王
         │
    男人 ─┼─ 女人
         │
        女王

向量关系：
国王 - 男人 + 女人 ≈ 女王

数学表达：
v("国王") - v("男人") + v("女人") ≈ v("女王")
```

| 词嵌入方法 | 特点 | 代表模型 |
|-----------|------|---------|
| **One-Hot** | 高维稀疏，无语义关系 | 基础编码 |
| **Word2Vec** | 稠密向量，捕获语义关系 | Skip-gram、CBOW |
| **GloVe** | 结合全局统计信息 | Stanford GloVe |
| **FastText** | 考虑子词信息 | Facebook FastText |

### 5.2 语言模型（Language Model）

语言模型计算一个句子出现的概率，或预测下一个词。

```
语言模型目标：

P("自然语言处理很有趣")
= P(自然) × P(语言|自然) × P(处理|自然语言) × P(很|自然语言处理) × P(有趣|自然语言处理很) × ...

或预测下一个词：

输入："自然语言"
预测：P(处理|自然语言) = 0.85
      P(学|自然语言) = 0.10
      P(的|自然语言) = 0.05
```

| 语言模型类型 | 特点 | 代表 |
|-------------|------|------|
| **N-gram** | 基于统计，考虑前N-1个词 | 传统统计LM |
| **神经LM** | 神经网络预测 | RNNLM |
| **预训练LM** | 大规模预训练，上下文相关 | BERT、GPT |

### 5.3 文本表示方法演进

```
文本表示演进：

Bag of Words ──→ TF-IDF ──→ Word2Vec ──→ ELMo ──→ BERT/GPT
(词袋)          (加权)       (静态词向量)  (动态词向量) (上下文表示)

特点变化：
- 维度：高维稀疏 → 低维稠密
- 语义：无 → 有
- 上下文：无关 → 相关
- 多义：无 → 有
```

## 六、NLP在AI产品中的应用

### 6.1 典型产品场景

| 产品类型 | NLP技术 | 代表产品 |
|---------|---------|---------|
| **智能客服** | 意图识别、对话管理、文本生成 | 阿里小蜜、腾讯小微 |
| **智能写作** | 文本生成、风格迁移 |  Jasper、Copy.ai |
| **智能搜索** | 语义理解、Query解析 | Google、百度 |
| **智能翻译** | 机器翻译 | DeepL、Google翻译 |
| **内容审核** | 文本分类、敏感词检测 | 各平台审核系统 |
| **知识图谱** | 信息抽取、关系抽取 | 百度知识图谱 |
| **智能助手** | 全栈NLP | Siri、小爱同学 |

### 6.2 AI产品经理的NLP关注点

```
AI产品经理的NLP知识边界：

需要理解：
✅ NLP任务类型（分类/生成/抽取/理解）
✅ 文本预处理流程
✅ 评估指标（准确率、BLEU、ROUGE等）
✅ 技术选型（规则/统计/深度学习/大模型）
✅ 应用场景和局限性

不需要深入：
❌ 具体模型架构实现
❌ 训练细节和超参调优
❌ 分词算法内部逻辑
```

### 6.3 NLP项目评估指标

| 任务 | 指标 | 说明 |
|------|------|------|
| **分类** | 准确率、精确率、召回率、F1 | 常规分类指标 |
| **翻译** | BLEU | n-gram精确率加权平均 |
| **摘要** | ROUGE | 召回率导向的n-gram匹配 |
| **生成** | Perplexity | 语言模型困惑度（越低越好） |
| **NER** | F1（实体级别） | 考虑边界和类型 |

## 七、本章小结

```
NLP核心知识框架：

NLP
│
├── 层次
│   ├── 词法（分词、词性、NER）
│   ├── 句法（依存分析、成分分析）
│   └── 语义（词义消歧、语义角色）
│
├── 任务
│   ├── 理解：分类、NER、意图识别
│   ├── 生成：翻译、摘要、对话
│   └── 抽取：实体、关系、事件
│
├── 技术演进
│   ├── 规则方法 → 统计方法
│   ├── 神经网络 → 预训练+微调
│   └── 大模型时代（Prompt工程）
│
└── 核心概念
    ├── 词嵌入（Word Embedding）
    ├── 语言模型（Language Model）
    └── 文本表示（从BoW到BERT）
```

**关键认知**：
1. NLP是AI产品的重要基础，几乎所有AI交互都涉及语言理解
2. 大模型时代，传统NLP pipeline被统一为"预训练+提示"
3. 分词和文本预处理是NLP的基础，直接影响模型效果
4. 评估NLP效果需要任务特定的指标，不能只看准确率
5. AI产品经理需要理解NLP能力边界，合理设计产品预期
