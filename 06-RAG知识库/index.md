# 06 - RAG 知识库

> 本章系统讲解 RAG（检索增强生成）技术体系，从基础概念、文档分块、文本向量化，到向量数据库选型、召回与重排序策略，最终落地企业级知识库设计，帮助你构建“让 AI 拥有私有知识”的核心能力。

---

## 本章定位

`06-RAG知识库` 解决的不是“会不会做一个文档问答 Demo”，而是以下 4 个更关键的问题：

1. **如何让大模型在企业私有知识上稳定回答，而不是只靠通用预训练记忆**
2. **如何把文档处理、检索、排序、生成串成可上线的知识链路**
3. **如何在准确率、延迟、成本、安全和可维护性之间做工程化平衡**
4. **如何把知识库能力真正映射到客服、搜索、Copilot、内部知识助手等业务场景**

## 企业级学习原则

1. **先理解全链路，再优化局部组件**：不要一上来只盯向量库或某个模型参数
2. **先做可验证闭环，再做复杂架构**：先跑通“数据接入 -> 分块 -> 检索 -> 回答 -> 评测”
3. **先明确业务目标，再谈技术选型**：RAG 不是默认最优解，需看场景与收益
4. **始终同时关注效果、成本和风险**：命中率高不等于业务可用，高可用也不等于合规

- [ ] 能识别企业知识库中的权限、安全、更新、成本和监控要求

## 关键术语速查

| 术语              | 说明                                                                          |
| ----------------- | ----------------------------------------------------------------------------- |
| **RAG**           | Retrieval-Augmented Generation，检索增强生成，通过外挂知识库增强 LLM 回答质量 |
| **Chunk**         | 文档分块，将长文档切分为适合 Embedding 处理的文本单元                         |
| **Embedding**     | 嵌入/向量化，将文本映射到高维稠密向量空间                                     |
| **Vector DB**     | 向量数据库，专门存储和检索高维向量的数据库                                    |
| **Recall**        | 召回，从知识库中检索相关文档的过程                                            |
| **Rerank**        | 重排序，对召回结果进行更精细的二次排序                                        |
| **Hybrid Search** | 混合搜索，融合稠密向量和稀疏向量（关键词）的检索策略                          |
| **HNSW**          | Hierarchical Navigable Small World，分层可导航小世界图索引算法                |
| **RRF**           | Reciprocal Rank Fusion，倒数排名融合，多路召回结果融合算法                    |
| **HyDE**          | Hypothetical Document Embeddings，假设文档嵌入，查询扩展技术                  |

## 学习建议

1. **先理解 RAG 全链路**：从 RAG基础开始，理解"为什么需要 RAG"和"RAG 怎么工作"
2. **重点掌握分块和 Embedding**：分块质量决定检索质量，Embedding 是检索的底层原理
3. **动手构建最小系统**：用 Chroma + OpenAI Embedding 快速构建一个 RAG 系统，理解全流程
4. **掌握检索优化策略**：Recall、Rerank、HybridSearch 是提升检索质量的关键技术
5. **选型向量数据库**：根据场景选择合适的向量库（Chroma/Milvus/Weaviate）
6. **设计企业级方案**：学习企业知识库设计，掌握系统架构、安全合规、运维监控
7. **持续优化迭代**：RAG 系统需要持续优化检索策略和分块方案，建立监控和反馈机制
