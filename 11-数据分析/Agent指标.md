# Agent指标

> Agent 产品最容易出现的一种数据误判，就是拿聊天产品或普通 AI 工具的指标去衡量它。比如只看 DAU、消息数、会话时长，往往根本看不出 Agent 到底有没有帮用户把任务做完。因为 Agent 的核心不是“说了多少话”，而是“是否围绕目标持续推进任务、是否在关键节点做出正确动作、是否在失败后可恢复、是否在可接受成本内交付结果”。所以，衡量 Agent 产品的重点，必须从“对话活跃”转到“任务完成”。本章的目标，就是帮你建立一套真正适用于 Agent 的指标视角。

## 一、为什么 Agent 不能只看活跃指标

### 1. 因为活跃不代表完成任务

一个 Agent 产品里，用户可能：

- 发起了很多任务
- 看到了很多中间反馈
- 进行了多轮交互

但这并不代表任务真的完成了。

如果只看：

- 会话数
- 用户时长
- 消息轮数

你可能会误把“复杂和低效”看成“高活跃”。

### 2. 因为 Agent 天然存在更长的执行链路

Agent 经常要经过：

- 目标理解
- 任务拆解
- 工具调用
- 中间判断
- 失败恢复
- 人工接管

这意味着只看最终结果也不够，你还要看过程是否健康。

### 3. 因为 Agent 的成本和风险被放大了

Agent 相比普通问答产品，往往会带来更高的：

- Token 成本
- 工具调用成本
- 失败重试成本
- 人工兜底成本

所以 Agent 指标必须更强调“值不值”和“稳不稳”。

---

## 二、Agent 指标最核心的关注点是什么

### 1. 任务有没有完成

这是 Agent 指标体系的第一层。

因为用户委托 Agent，本质上是希望它帮自己推进目标，而不是只给几个建议。

### 2. 任务是怎么完成的

即使最终完成，也要看：

- 是否走了很多无效步骤
- 是否频繁失败再重试
- 是否大量依赖人工介入

### 3. 完成过程是否可控

企业场景里特别重要的是：

- 是否有越权执行
- 是否有高风险动作
- 是否能回溯执行链
- 是否能在异常时停止

### 4. 完成的代价是否合理

同样完成一个任务，不同 Agent 方案可能有完全不同的：

- 成本
- 时长
- 人工消耗
- 风险暴露

所以 Agent 不是“能完成就行”，而是要看完成得是否值得。

---

## 三、Agent 指标和普通 AI 产品指标的最大差异

### 1. 从“单次响应”转向“任务闭环”

普通 AI 产品常常更关注：

- 一次回答质量
- 一次建议采纳率
- 一次会话体验

Agent 更应该关注：

- 一个任务是否从目标到结果形成闭环

### 2. 从“内容质量”扩展到“执行质量”

Agent 的质量不只在于说得对不对，还在于：

- 调的工具对不对
- 顺序对不对
- 是否在错误时及时止损

### 3. 从“用户交互”扩展到“系统行动”

普通 AI 产品很多时候只分析用户行为。

Agent 产品必须同时分析：

- Agent 自己执行了什么
- 哪一步失败了
- 哪一类动作最不稳定

### 4. 从“体验指标”扩展到“协作指标”

Agent 里很重要的一类指标是：

- 人工接管率
- 人工确认率
- 人工修改距离

这些指标反映的不是单纯体验，而是人机协作效率。

---

## 四、Agent 指标的六层结构

### 1. 任务结果指标

这一层回答：

- Agent 最终有没有帮用户完成目标

常见指标包括：

- 任务完成率
- 有效完成率
- 任务放弃率
- 任务失败率

### 2. 阶段推进指标

这一层回答：

- Agent 卡在了哪一步

常见指标包括：

- 阶段完成率
- 规划成功率
- 中途退出率
- 阶段耗时

### 3. 工具调用指标

这一层回答：

- Agent 的行动链是否稳定

常见指标包括：

- 工具调用成功率
- 工具超时率
- 工具重试率
- 错误工具调用率

### 4. 人机协作指标

这一层回答：

- 人和 Agent 的协作是否顺畅

常见指标包括：

- 人工接管率
- 关键节点确认率
- 人工修改距离
- 人工复核通过率

### 5. 成本指标

这一层回答：

- Agent 的执行是否经济

常见指标包括：

- 单任务 Token 成本
- 单任务工具成本
- 单任务人工补救成本
- 总任务成本

### 6. 风险指标

这一层回答：

- Agent 有没有在制造不可接受的风险

常见指标包括：

- 高风险动作触发率
- 越权尝试率
- 回滚率
- 敏感任务失败率

---

## 五、任务完成率为什么是 Agent 的第一核心指标

### 1. 因为 Agent 本质上是任务型产品

Agent 和聊天产品的最大不同是：

- 用户是来委托任务的

所以如果不能完成任务，再高的会话时长也没有太大意义。

### 2. 但“任务完成率”不能定义得太粗

需要进一步区分：

- 形式完成
- 真正有效完成

例如：

- 生成了报告不代表报告可用
- 建了工单不代表问题被正确归类

### 3. 更推荐的做法

可以拆成：

- 任务启动率
- 阶段完成率
- 最终完成率
- 有效完成率

### 4. 企业里尤其要重视“有效完成率”

因为企业真正关心的不是：

- Agent 有没有输出

而是：

- 输出是否能进入业务流程
- 是否减少了人工工作量
- 是否没有带来额外风险

---

## 六、阶段完成率和路径分析，为什么比普通产品更重要

### 1. Agent 经常不是一步到位

它通常会经历多个阶段，例如：

1. 理解目标
2. 制定计划
3. 检索资料
4. 调用工具
5. 输出结果

### 2. 只看最终完成率会掩盖很多问题

比如你只看到完成率下降，但不知道究竟是：

- 规划失败
- 工具故障
- 任务目标不清
- 用户在关键节点不愿确认

### 3. 阶段指标可以帮助快速定位问题

例如：

- 目标确认率低，说明用户输入设计有问题
- 工具调用成功率低，说明系统链路不稳定
- 最终确认通过率低，说明结果质量不足

### 4. 对 Agent 来说，路径分析比聊天产品更有解释力

因为 Agent 产品的数据问题，很多都发生在“任务过程”里，而不是最终呈现层。

---

## 七、工具调用指标为什么是 Agent 专属重点

### 1. Agent 的能力很多来自工具，不只来自模型

Agent 的价值经常依赖：

- 检索工具
- 数据库工具
- 工单工具
- 日程工具
- 文档工具

所以工具链健康度几乎决定了 Agent 的下限。

### 2. 常见需要重点看的工具指标

包括：

- 调用成功率
- 超时率
- 平均耗时
- 重试次数
- 工具结果可用率

### 3. 工具指标如何帮助解释 Agent 问题

例如：

- 任务完成率下降，但模型表现没变
- 进一步发现数据库工具超时率大幅上升

这时就能知道问题主要不在模型，而在工具链。

### 4. 产品经理为什么也要看工具指标

因为很多 Agent 产品失败，看起来像“AI 不行”，实际是：

- 工具接得太乱
- 写操作风险太高
- 返回结构不稳定

这些都直接影响产品体验和业务价值。

---

## 八、人机接管率为什么不能简单理解为越低越好

### 1. 很多人会误以为接管率越低，Agent 越强

这并不总是对的。

在高风险或复杂场景中，合理接管本身就是健康策略。

### 2. 应该怎么看待接管率

更合理的方式是分场景看：

- 低风险任务中接管率过高，说明 Agent 不够成熟
- 高风险任务中接管率适度，反而说明边界感清楚

### 3. 接管率需要和什么一起看

建议同时看：

- 任务类型
- 风险等级
- 接管原因
- 接管后的成功率

### 4. 真正更重要的不是“是否接管”，而是“接管是否及时且有效”

如果 Agent 在该停的时候不停，风险会比高一点的接管率更严重。

---

## 九、Agent 的成本指标为什么必须细到“单任务”

### 1. 总成本不够解释问题

只看总成本，你很难知道：

- 是因为用户多了
- 还是单次任务太贵了

### 2. 更有价值的是单任务成本

例如：

- 单任务 Token 成本
- 单任务工具成本
- 单任务人工补救成本

这些能更直观看出：

- 当前 Agent 是否具备规模化可能

### 3. 单任务成本为什么在 Agent 场景里更敏感

因为 Agent 往往：

- 步骤更多
- 链路更长
- 重试更频繁
- 工具更多

所以稍微失控，成本就会迅速放大。

### 4. 产品经理如何用成本指标做判断

常见问题包括：

- 任务完成率提升是否值得这部分新增成本
- 某类复杂任务是否应改成人工优先
- 是否应对高成本低价值任务做范围收缩

---

## 十、Agent 风险指标该怎么定义

### 1. Agent 的风险往往来自行动，而不是回答

常见风险包括：

- 调错工具
- 写错数据
- 误发通知
- 在错误路径上继续执行

### 2. 常见风险指标

可以重点看：

- 高风险动作触发率
- 越权尝试率
- 回滚率
- 敏感任务失败率
- 违规操作拦截率

### 3. 为什么这些指标重要

因为企业通常不会容忍：

- 为了更高完成率而放大业务风险

### 4. 风险指标不只是安全团队关心

产品经理也必须关注，因为这些指标会直接决定：

- 上线范围
- 灰度策略
- 是否允许从试点走向规模化

---

## 十一、不同 Agent 场景的核心指标会怎么变化

### 1. 信息整理型 Agent

更适合关注：

- 任务完成率
- 结果可用率
- 人工修改距离
- 单任务成本

### 2. 执行型 Agent

更适合关注：

- 执行成功率
- 关键动作确认率
- 回滚率
- 风险事件率

### 3. 研究分析型 Agent

更适合关注：

- 阶段完成率
- 资料覆盖率
- 结果结构完整度
- 人工复核通过率

### 4. 协作型 Agent

更适合关注：

- 任务交接成功率
- 协作节点确认率
- 人工接管时效
- 任务闭环率

所以 Agent 指标并不是一张固定清单，而是要结合任务属性来设计。

## 十二、一个完整案例：售前方案 Agent 该怎么看指标

### 1. 场景背景

某团队做了一个售前方案 Agent，希望它能：

- 理解客户需求
- 检索历史案例
- 汇总产品能力
- 生成初版方案草稿

### 2. 如果只看“用户是否使用”，会有什么问题

可能会看到：

- 使用人数在增长
- 提交任务数也不少

但你仍然不知道：

- 草稿能不能用
- 中间是不是反复失败
- 是否大量依赖人工修订

### 3. 更合理的指标体系

建议重点看：

- 任务启动率
- 阶段完成率
- 最终有效完成率
- 工具调用成功率
- 人工修改距离
- 单任务成本

### 4. 从这些指标可能得出什么结论

例如：

- 启动率高但有效完成率低，说明用户愿意尝试但结果价值不够
- 工具调用成功率低，说明问题可能在系统集成层
- 完成率还行但人工修改距离很大，说明 Agent 更像“半成品生成器”

这类分析比单纯看使用量更接近产品真相。

## 十三、Agent 看板应该怎么设计

### 1. 第一层放结果指标

例如：

- 任务完成率
- 有效完成率
- 任务放弃率

### 2. 第二层放阶段推进指标

例如：

- 目标确认率
- 规划完成率
- 工具执行阶段成功率
- 最终输出通过率

### 3. 第三层放工具与协作指标

例如：

- 工具成功率
- 工具超时率
- 人工接管率
- 接管原因分布

### 4. 第四层放成本与风险

例如：

- 单任务成本
- 平均执行时长
- 回滚率
- 风险事件率

Agent 看板的目标，不是展示所有细节，而是帮助团队快速判断：

- 值不值得继续扩场景
- 哪一段链路最拖后腿
- 哪一类任务应收缩边界

## 十五、常见误区补充

### 误区 1：Agent 的指标和聊天产品差不多

错误。Agent 更应围绕任务闭环、工具链和人机协作来设计指标。

### 误区 2：任务完成率高就说明 Agent 很强

错误。还要看完成是否有效、是否依赖大量人工、成本是否合理。

### 误区 3：人工接管率越低越好

错误。关键是接管是否发生在该发生的时候，以及接管后是否顺畅。

### 误区 4：工具指标属于技术问题，不属于产品问题

错误。工具调用稳定性直接决定 Agent 是否有真实产品价值。

### 误区 5：总成本可控就行，不需要看单任务成本

错误。Agent 能否规模化，很大程度上取决于单任务成本结构。

## 十六、本章小结

如果用一句话总结：

**Agent 指标体系的核心，不是衡量它说了多少，而是衡量它是否在可控边界内，以合理成本真实完成了任务。**

对 AI 产品经理来说，本章最重要的收获包括：

- 学会用任务闭环视角而不是对话视角看 Agent
- 学会同时衡量结果、阶段、工具、人机协作、成本和风险
- 学会用指标识别 Agent 是“真有价值”还是“看起来很忙”
