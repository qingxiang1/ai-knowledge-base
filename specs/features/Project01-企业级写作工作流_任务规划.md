# Project01 企业级写作工作流任务规划

## 阶段总览

```mermaid
flowchart LR
  T1[Task-01 类型与状态模型] --> T2[Task-02 后端状态机与接口]
  T2 --> T3[Task-03 前端 API 与 Hook]
  T3 --> T4[Task-04 编辑区工作台]
  T3 --> T5[Task-05 侧边栏与审计轨迹]
  T4 --> T6[Task-06 联调与类型检查]
  T5 --> T6
```

## 任务清单

- [x] Task-01 类型与状态模型
  - 通俗解释：先把“企业草稿是什么、有哪些状态、谁能做什么”定义清楚。
  - 验证标准：前后端都存在统一的角色、状态、单据和审计日志类型定义，可支持草稿、审核、发布三段流程。
  - 对应 AC：AC-001, AC-009, AC-010
  - 预计工时：40 分钟

- [x] Task-02 后端状态机与接口
  - 通俗解释：让服务器真正知道什么时候能提交审核、什么时候能通过、什么时候能发布。
  - 验证标准：6 个接口可正确返回成功或错误信息；非法角色和非法状态会被拦截；所有状态变更产生审计日志。
  - 对应 AC：AC-001, AC-002, AC-003, AC-004, AC-006, AC-007, AC-008, AC-009, AC-010
  - 预计工时：90 分钟

- [x] Task-03 前端 API 与 Hook
  - 通俗解释：让页面能拿到模板和单据，并把每个按钮操作真正发到后端。
  - 验证标准：前端可以加载模板和单据；点击生成、提交审核、审核、发布后页面状态同步更新；错误信息可见。
  - 对应 AC：AC-001 至 AC-010
  - 预计工时：70 分钟

- [x] Task-04 编辑区工作台
  - 通俗解释：把作者真正需要填写和点击的区域搭起来。
  - 验证标准：用户可以填写标题、模板、摘要并生成草稿；不同角色看到的按钮符合状态机约束。
  - 对应 AC：AC-001, AC-002, AC-003, AC-004, AC-005, AC-006, AC-007, AC-008
  - 预计工时：70 分钟

- [x] Task-05 侧边栏与审计轨迹
  - 通俗解释：让用户一眼看见当前有哪些内容单据、它们现在走到哪一步了。
  - 验证标准：单据列表可切换；当前状态和版本清晰可见；审计日志展示动作、时间、角色和备注。
  - 对应 AC：AC-009, AC-010
  - 预计工时：50 分钟

- [x] Task-06 联调与类型检查
  - 通俗解释：把所有页面和接口接成一条完整流程，并确保代码可正常构建。
  - 验证标准：前端和后端 TypeScript 检查通过；完整流程可手工走通；无明显类型错误。
  - 对应 AC：AC-001 至 AC-010
  - 预计工时：40 分钟
