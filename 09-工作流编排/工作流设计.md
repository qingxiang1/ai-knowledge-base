## 一、工作流设计原则

### 1.1 核心设计原则

```
AI 工作流设计原则：

原则一：任务分解
├── 将复杂任务拆分为原子操作
├── 每个节点职责单一
├── 节点间通过明确接口通信
└── 示例：
    内容生成工作流
    ├── 需求分析节点
    ├── 大纲生成节点
    ├── 内容撰写节点
    ├── 质量检查节点
    └── 格式化输出节点

原则二：失败隔离
├── 单个节点失败不影响整体
├── 实现错误重试机制
├── 设置超时和降级策略
└── 示例：
    ├── 主路径：正常执行
    ├── 错误路径：记录日志 + 通知
    └── 降级路径：使用缓存结果

原则三：数据驱动
├── 工作流状态可观测
├── 输入输出数据可追溯
├── 支持中间结果检查
└── 示例：
    ├── 每个节点记录处理时间
    ├── 记录输入数据摘要
    └── 记录输出数据质量

原则四：弹性扩展
├── 支持水平扩展
├── 资源使用可配置
├── 负载自动调节
└── 示例：
    ├── 并发数根据负载调整
    ├── 模型选择根据成本调整
    └── 缓存策略根据命中率调整

原则五：人机协同
├── 关键决策点保留人工确认
├── 异常处理需要人工介入
├── 持续学习人工反馈
└── 示例：
    ├── 高风险操作需要审批
    ├── 低置信度结果需要确认
    └── 定期人工审核输出质量
```

### 1.2 设计模式

```python
"""
AI 工作流设计模式

常见的工作流架构模式
"""

from typing import Dict, List, Callable, Any
from dataclasses import dataclass
from enum import Enum

class WorkflowPattern(Enum):
    """工作流模式枚举"""
    SEQUENTIAL = "sequential"
    PARALLEL = "parallel"
    CONDITIONAL = "conditional"
    ITERATIVE = "iterative"
    MAP_REDUCE = "map_reduce"
    FAN_OUT_IN = "fan_out_in"
    SAGA = "saga"

@dataclass
class WorkflowNode:
    """工作流节点"""
    id: str
    name: str
    execute: Callable
    retry_count: int = 3
    timeout: int = 30
    fallback: Callable = None

class WorkflowPatterns:
    """工作流模式实现"""
    
    @staticmethod
    def sequential(nodes: List[WorkflowNode], input_data: Any) -> Any:
        """
        顺序执行模式
        
        节点依次执行，前一个节点的输出作为后一个节点的输入
        
        Args:
            nodes: 节点列表
            input_data: 初始输入
        
        Returns:
            最终结果
        """
        result = input_data
        for node in nodes:
            try:
                result = node.execute(result)
            except Exception as e:
                if node.fallback:
                    result = node.fallback(result, e)
                else:
                    raise
        return result
    
    @staticmethod
    def parallel(nodes: List[WorkflowNode], input_data: Any) -> List[Any]:
        """
        并行执行模式
        
        多个节点同时执行，结果合并返回
        
        Args:
            nodes: 节点列表
            input_data: 初始输入
        
        Returns:
            所有节点的结果列表
        """
        from concurrent.futures import ThreadPoolExecutor
        
        def execute_node(node: WorkflowNode):
            try:
                return node.execute(input_data)
            except Exception as e:
                if node.fallback:
                    return node.fallback(input_data, e)
                raise
        
        with ThreadPoolExecutor() as executor:
            results = list(executor.map(execute_node, nodes))
        
        return results
    
    @staticmethod
    def conditional(node: WorkflowNode, condition: Callable,
                   true_branch: List[WorkflowNode],
                   false_branch: List[WorkflowNode],
                   input_data: Any) -> Any:
        """
        条件分支模式
        
        根据条件选择执行路径
        
        Args:
            node: 条件判断节点
            condition: 条件函数
            true_branch: 真分支
            false_branch: 假分支
            input_data: 初始输入
        
        Returns:
            执行结果
        """
        condition_result = condition(input_data)
        
        if condition_result:
            return WorkflowPatterns.sequential(true_branch, input_data)
        else:
            return WorkflowPatterns.sequential(false_branch, input_data)
    
    @staticmethod
    def iterative(node: WorkflowNode, evaluator: Callable,
                 max_iterations: int = 5,
                 input_data: Any = None) -> Any:
        """
        迭代优化模式
        
        重复执行直到满足条件或达到最大迭代次数
        
        Args:
            node: 执行节点
            evaluator: 评估函数
            max_iterations: 最大迭代次数
            input_data: 初始输入
        
        Returns:
            最终结果
        """
        result = input_data
        
        for i in range(max_iterations):
            result = node.execute(result)
            
            if evaluator(result):
                break
        
        return result
    
    @staticmethod
    def map_reduce(map_nodes: List[WorkflowNode],
                  reduce_node: WorkflowNode,
                  input_data: List[Any]) -> Any:
        """
        Map-Reduce 模式
        
        分批处理，然后聚合结果
        
        Args:
            map_nodes: 映射节点列表
            reduce_node: 聚合节点
            input_data: 输入数据列表
        
        Returns:
            聚合结果
        """
        # Map 阶段
        mapped_results = []
        for item in input_data:
            for node in map_nodes:
                item = node.execute(item)
            mapped_results.append(item)
        
        # Reduce 阶段
        return reduce_node.execute(mapped_results)

# 模式选择指南
"""
模式选择指南：

1. 顺序模式（Sequential）
   ├── 适用：有依赖关系的任务
   ├── 示例：数据清洗 → 特征提取 → 模型推理
   └── 注意：总耗时 = 各节点耗时之和

2. 并行模式（Parallel）
   ├── 适用：无依赖的独立任务
   ├── 示例：同时调用多个模型
   └── 注意：总耗时 = 最慢节点耗时

3. 条件模式（Conditional）
   ├── 适用：需要根据中间结果决策
   ├── 示例：质量检查 → 通过/不通过
   └── 注意：确保条件判断准确

4. 迭代模式（Iterative）
   ├── 适用：需要逐步优化的任务
   ├── 示例：内容生成 → 评估 → 改进
   └── 注意：设置最大迭代次数防止死循环

5. Map-Reduce 模式
   ├── 适用：大数据量分批处理
   ├── 示例：批量文档处理
   └── 注意：Reduce 节点需要能处理聚合逻辑
"""
```

```python
"""

from typing import Dict, List
from dataclasses import dataclass
from enum import Enum

class EvaluationDimension(Enum):
    """评估维度"""
    ACCURACY = "accuracy"
    LATENCY = "latency"
    COST = "cost"
    RELIABILITY = "reliability"
    SCALABILITY = "scalability"
    MAINTAINABILITY = "maintainability"

@dataclass
class EvaluationMetric:
    """评估指标"""
    dimension: EvaluationDimension
    metric_name: str
    target_value: float
    actual_value: float
    weight: float = 1.0

class WorkflowEvaluator:
    """工作流评估器"""
    
    def __init__(self):
        """初始化评估器"""
        self.metrics = []
    
    def add_metric(self, metric: EvaluationMetric):
        """
        添加评估指标
        
        Args:
            metric: 评估指标
        """
        self.metrics.append(metric)
    
    def evaluate(self) -> Dict:
        """
        执行评估
        
        Returns:
            评估结果
        """
        total_score = 0
        total_weight = 0
        details = []
        
        for metric in self.metrics:
            # 计算得分（实际值/目标值，上限为1）
            score = min(metric.actual_value / metric.target_value, 1.0)
            weighted_score = score * metric.weight
            
            total_score += weighted_score
            total_weight += metric.weight
            
            details.append({
                "dimension": metric.dimension.value,
                "metric": metric.metric_name,
                "target": metric.target_value,
                "actual": metric.actual_value,
                "score": score,
                "weighted_score": weighted_score
            })
        
        overall_score = total_score / total_weight if total_weight > 0 else 0
        
        return {
            "overall_score": overall_score,
            "details": details,
            "grade": self._get_grade(overall_score)
        }

        获取等级
```

### 3.2 错误处理

```python
"""
工作流错误处理策略

构建健壮的错误处理机制
"""

from typing import Callable, Any
from enum import Enum
import logging

class ErrorStrategy(Enum):
    """错误处理策略"""
    RETRY = "retry"
    FALLBACK = "fallback"
    CIRCUIT_BREAKER = "circuit_breaker"
    DEAD_LETTER = "dead_letter"
    IGNORE = "ignore"

class ErrorHandler:
    """错误处理器"""
    
    def __init__(self):
        """初始化错误处理器"""
        self.logger = logging.getLogger(__name__)
        self.circuit_states = {}
    
    def handle(self, error: Exception, strategy: ErrorStrategy,
              context: dict = None) -> Any:
        """
        处理错误
        
        Args:
            error: 异常
            strategy: 处理策略
            context: 上下文信息
        
        Returns:
            处理结果
        """
        self.logger.error(f"Error occurred: {error}", extra=context)
        
        if strategy == ErrorStrategy.RETRY:
            return self._retry_handler(error, context)
        elif strategy == ErrorStrategy.FALLBACK:
            return self._fallback_handler(error, context)
        elif strategy == ErrorStrategy.CIRCUIT_BREAKER:
            return self._circuit_breaker_handler(error, context)
        elif strategy == ErrorStrategy.DEAD_LETTER:
            return self._dead_letter_handler(error, context)
        elif strategy == ErrorStrategy.IGNORE:
            return None
        else:
            raise error
    
    def _retry_handler(self, error: Exception, context: dict) -> Any:
        """
        重试处理
        
        指数退避重试
        """
        import time
        
        max_retries = context.get("max_retries", 3)
        base_delay = context.get("base_delay", 1)
        
        for attempt in range(max_retries):
            try:
                delay = base_delay * (2 ** attempt)
                time.sleep(delay)
                
                # 重试执行
                operation = context.get("operation")
                if operation:
                    return operation()
            except Exception as e:
                self.logger.warning(f"Retry {attempt + 1} failed: {e}")
                if attempt == max_retries - 1:
                    raise
    
    def _fallback_handler(self, error: Exception, context: dict) -> Any:
        """
        降级处理
        
        使用备用方案
        """
        fallback = context.get("fallback")
        if fallback:
            self.logger.info("Using fallback strategy")
            return fallback()
        else:
            raise error
    
    def _circuit_breaker_handler(self, error: Exception, context: dict) -> Any:
        """
        熔断处理
        
        防止级联故障
        """
        service_name = context.get("service_name", "default")
        
        if service_name not in self.circuit_states:
            self.circuit_states[service_name] = {
                "failures": 0,
                "threshold": context.get("threshold", 5),
                "timeout": context.get("timeout", 60),
                "last_failure": None,
                "state": "closed"  # closed, open, half-open
            }
        
        state = self.circuit_states[service_name]
        
        if state["state"] == "open":
            if time.time() - state["last_failure"] > state["timeout"]:
                state["state"] = "half-open"
            else:
                raise Exception(f"Circuit breaker is open for {service_name}")
        
        try:
            operation = context.get("operation")
            if operation:
                result = operation()
                state["failures"] = 0
                state["state"] = "closed"
                return result
        except Exception as e:
            state["failures"] += 1
            state["last_failure"] = time.time()
            
            if state["failures"] >= state["threshold"]:
                state["state"] = "open"
            
            raise
    
    def _dead_letter_handler(self, error: Exception, context: dict) -> Any:
        """
        死信队列处理
        
        记录失败任务供后续处理
        """
        dead_letter_queue = context.get("dead_letter_queue", [])
        dead_letter_queue.append({
            "error": str(error),
            "context": context,
            "timestamp": time.time()
        })
        
        self.logger.info("Message sent to dead letter queue")
        return None

# 错误处理最佳实践
"""
错误处理最佳实践：

1. 分层处理
   ├── 节点级别：重试 + 降级
   ├── 工作流级别：补偿 + 回滚
   └── 系统级别：熔断 + 限流

2. 监控告警
   ├── 错误率监控
   ├── 响应时间监控
   └── 业务指标监控

3. 日志记录
   ├── 结构化日志
   ├── 上下文信息
   └── 链路追踪

4. 人工介入
   ├── 严重错误通知
   ├── 批量失败告警
   └── 定期人工审核
```
