# Agent面试题

> Agent技术面试题，覆盖任务规划、工具调用、记忆管理与多Agent协作。

---

## 一、Agent基础

### 1.1 核心概念

**Q1: 什么是AI Agent？与Chatbot有什么区别？**

```
参考答案要点：
- Agent定义：
  - 能感知环境、自主决策、执行行动的AI系统
  - 核心能力：规划、记忆、工具使用、反思

- Chatbot区别：
  - Chatbot：被动响应，单轮/多轮对话
  - Agent：主动执行，可调用工具完成任务
  - Agent是Chatbot的进化形态

- 关键特征：
  - 自主性：无需人工逐步指令
  - 工具使用：能调用外部API/工具
  - 记忆：能记住上下文和经验
  - 反思：能评估并优化行动
```

**Q2: Agent的核心架构组件有哪些？**

```
参考答案要点：
- 感知（Perception）：
  - 接收用户输入
  - 感知环境状态
  - 理解任务目标

- 规划（Planning）：
  - 任务分解
  - 制定执行计划
  - 选择策略

- 记忆（Memory）：
  - 短期记忆：当前上下文
  - 长期记忆：历史经验
  - 工作记忆：任务状态

- 行动（Action）：
  - 工具调用
  - 代码执行
  - 结果输出

- 反思（Reflection）：
  - 评估执行结果
  - 识别错误
  - 优化策略
```

### 1.2 规划策略

**Q3: Agent的任务规划有哪些方法？**

```
参考答案要点：
- ReAct（Reasoning + Acting）：
  - 交替进行推理和行动
  - Thought → Action → Observation
  - 适合需要逐步推理的任务

- CoT（Chain of Thought）：
  - 思维链提示
  - 分解问题→逐步推理→得出答案
  - 适合数学、逻辑推理

- ToT（Tree of Thoughts）：
  - 思维树搜索
  - 多路径探索→评估→选择最优
  - 适合复杂决策

- Plan-and-Solve：
  - 先规划后执行
  - 生成计划→按步骤执行
  - 适合明确步骤的任务
```

**Q4: 什么是ReAct框架？请举例说明**

```
参考答案要点：
- 框架原理：
  - 将推理（Reasoning）和行动（Acting）结合
  - 每步先思考，再行动，再观察

- 执行流程：
  Thought: 我需要查询北京的天气
  Action: 调用天气API(query="北京")
  Observation: 北京今天晴，25°C
  Thought: 用户问是否需要带伞
  Action: 生成回答("不需要带伞，今天晴天")

- 优势：
  - 可解释性强
  - 错误可追踪
  - 适合复杂任务
```

## 二、工具调用

### 2.1 工具设计

**Q5: Agent如何设计和使用工具？**

```
参考答案要点：
- 工具定义：
  - 名称：工具标识
  - 描述：功能说明
  - 参数：输入参数定义
  - 返回值：输出格式

- 工具类型：
  - 搜索工具：网页搜索、知识检索
  - 计算工具：数学计算、代码执行
  - API工具：调用外部服务
  - 数据库工具：数据查询

- 调用方式：
  - Function Calling：模型输出函数调用
  - Toolformer：模型学习使用工具
  - 手动注册：预定义工具列表
```

**Q6: Function Calling的原理是什么？**

```
参考答案要点：
- 原理：
  - 模型学习识别何时需要调用工具
  - 输出结构化函数调用请求
  - 系统执行函数并返回结果

- 流程：
  1. 用户提问
  2. 模型判断需要工具
  3. 模型输出函数调用（JSON格式）
  4. 系统执行函数
  5. 结果返回给模型
  6. 模型生成最终回答

- 示例：
  用户："北京天气怎么样？"
  模型输出：{"function": "get_weather", "arguments": {"city": "北京"}}
  系统执行：返回天气数据
  模型回答："北京今天晴，25°C"
```

### 2.2 工具管理

**Q7: 如何实现Agent的动态工具发现？**

```
参考答案要点：
- 静态注册：
  - 启动时加载预定义工具
  - 适合固定功能场景

- 动态发现：
  - 扫描工具目录
  - 读取工具描述文件
  - 运行时注册

- 自动学习：
  - 从文档学习工具用法
  - 通过示例学习
  - 试错学习

- 权限控制：
  - 按用户分配工具
  - 按角色分配工具
  - 敏感工具需确认
```

## 三、记忆管理

### 3.1 记忆类型

**Q8: Agent的记忆系统如何设计？**

```
参考答案要点：
- 短期记忆：
  - 范围：当前会话
  - 存储：模型上下文窗口
  - 管理：滑动窗口、摘要压缩
  - 限制：受模型上下文长度限制

- 长期记忆：
  - 范围：跨会话
  - 存储：向量数据库
  - 管理：RAG检索、定期整理
  - 容量：理论上无限

- 工作记忆：
  - 范围：当前任务
  - 存储：内存变量
  - 管理：任务完成即清理
```

**Q9: 如何处理Agent的上下文长度限制？**

```
参考答案要点：
- 摘要压缩：
  - 对历史对话生成摘要
  - 保留关键信息
  - 丢弃细节

- 分层记忆：
  - 近期完整记录
  - 中期摘要记录
  - 长期索引记录

- 选择性保留：
  - 保留重要信息
  - 丢弃无关内容
  - 基于重要性评分

- 外部存储：
  - 将记忆存储到数据库
  - 需要时检索
  - 类似操作系统的虚拟内存
```

### 3.2 记忆操作

**Q10: Agent的记忆如何存储和检索？**

```
参考答案要点：
- 存储方式：
  - 即时存储：关键信息实时写入
  - 批量存储：定时批量写入
  - 触发存储：特定事件触发

- 检索方式：
  - 语义检索：向量相似度
  - 关键词检索：精确匹配
  - 时间检索：按时间范围
  - 关联检索：相关记忆联想

- 记忆整理：
  - 去重：合并相似记忆
  - 摘要：提取关键信息
  - 遗忘：删除低价值记忆
```

## 四、多Agent协作

### 4.1 协作模式

**Q11: 多Agent系统有哪些协作模式？**

```
参考答案要点：
- 层级协作：
  - Manager + Workers
  - Manager分配任务
  - Workers执行并汇报
  - 适合任务可分解

- 对等协作：
  - Peer-to-Peer
  - Agent间协商
  - 分工执行
  - 适合需要讨论决策

- 流水线：
  - Pipeline模式
  - Agent1输出→Agent2输入
  - 适合有明确依赖链

- 竞争协作：
  - 多Agent并行
  - 评估选择最优
  - 适合创意生成
```

**Q12: 如何设计多Agent的通信机制？**

```
参考答案要点：
- 通信方式：
  - 消息队列：异步通信
  - RPC：同步调用
  - 共享内存：高速共享
  - 黑板系统：共享工作区

- 消息格式：
  - 标准格式：JSON/Protobuf
  - 内容：角色、意图、数据
  - 元数据：时间戳、优先级

- 协调机制：
  - 中央协调器
  - 分布式共识
  - 自组织协商
```

### 4.2 协作挑战

**Q13: 多Agent系统面临哪些挑战？**

```
参考答案要点：
- 协调挑战：
  - 任务分配公平性
  - 避免重复工作
  - 处理依赖关系

- 一致性挑战：
  - 信息同步
  - 状态一致
  - 冲突解决

- 效率挑战：
  - 通信开销
  - 等待时间
  - 资源竞争

- 容错挑战：
  - 单点故障
  - 任务重分配
  - 结果合并
```

## 五、工程落地

### 5.1 架构设计

**Q14: Agent系统的工程架构如何设计？**

```
参考答案要点：
- 核心组件：
  - 规划器：任务分解与调度
  - 执行器：工具调用与执行
  - 记忆模块：记忆存储与检索
  - 反思器：结果评估与优化

- 数据流：
  - 输入→理解→规划→执行→观察→反思→输出

- 扩展性：
  - 插件化工具
  - 可替换规划策略
  - 可配置记忆后端

- 监控：
  - 执行轨迹记录
  - 工具调用统计
  - 错误告警
```

**Q15: 如何保证Agent系统的安全性？**

```
参考答案要点：
- 工具安全：
  - 工具权限控制
  - 敏感操作确认
  - 工具调用审计

- 数据安全：
  - 数据访问控制
  - 敏感信息脱敏
  - 数据隔离

- 执行安全：
  - 沙箱执行
  - 超时控制
  - 资源限制

- 内容安全：
  - 输入过滤
  - 输出审核
  - 合规检查
```

### 5.2 效果评估

**Q16: 如何评估Agent系统的效果？**

```
参考答案要点：
- 任务完成度：
  - 任务成功率
  - 完成时间
  - 步骤效率

- 工具使用：
  - 工具选择准确性
  - 参数填写正确率
  - 工具调用次数

- 用户体验：
  - 交互轮数
  - 用户满意度
  - 人工介入率

- 成本效益：
  - Token消耗
  - API调用成本
  - 执行时间
```
