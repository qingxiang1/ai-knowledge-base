# 年度学习计划

> 以12个月为周期，系统构建AI产品经理核心能力体系

---

## 一、计划概述

本计划将全年划分为四个阶段，每个阶段聚焦不同能力维度：

| 季度 | 主题   | 核心目标            | 关键产出                        |
| ---- | ------ | ------------------- | ------------------------------- |
| Q1   | 筑基期 | 建立产品+AI双重基础 | PRD文档、Prompt库、技术概念笔记 |
| Q2   | 进阶期 | 掌握AI核心应用能力  | RAG方案、Agent设计、工作流搭建  |
| Q3   | 实战期 | 完成完整项目实战    | 2个上线项目、作品集             |
| Q4   | 突破期 | 形成差异化竞争力    | 系统设计能力、行业影响力        |

**每日建议投入时间**：工作日2小时 + 周末4-6小时

---

### 1.1 年度执行原则

#### 1.1.1 原则一：全年只围绕 4 条主线推进

1. **岗位认知主线**：知道 AI 产品经理到底在解决什么问题
2. **能力建设主线**：补齐产品、技术、交付、商业四类能力
3. **项目产出主线**：把学习转成方案、Demo、项目和作品集
4. **求职/转岗主线**：从一开始就为简历、面试、内部转岗准备材料

#### 1.1.2 原则二：每个季度都必须有“可展示结果”

| 季度 | 不能缺少的产出                                     |
| ---- | -------------------------------------------------- |
| Q1   | 1 份完整 PRD + 1 套 Prompt 资产 + 1 份 AI 概念笔记 |
| Q2   | 1 份 RAG 方案 + 1 份 Agent 方案 + 1 个可运行 Demo  |
| Q3   | 2 个完整项目案例 + 作品集初版                      |
| Q4   | 1 份系统设计方案 + 1 份商业化分析 + 面试材料合集   |

#### 1.1.3 原则三：优先做“能证明自己”的事

建议把全年时间分配成如下结构：

| 类型     | 占比 | 说明                             |
| -------- | ---- | -------------------------------- |
| 系统学习 | 35%  | 阅读知识库、课程、书籍           |
| 动手实践 | 35%  | Prompt 设计、搭建 Demo、设计方案 |
| 项目沉淀 | 20%  | 文档、作品集、复盘材料           |
| 行业跟踪 | 10%  | 竞品体验、趋势观察、案例拆解     |

---

### 1.2 季度闸门与验收方式

> 把一年拆成 4 个“闸门”。每过一个季度，都要判断自己是否真正进入下一阶段，而不是只把计划打勾。

| 季度闸门 | 是否通过的核心判断                     | 推荐验收方式                       |
| -------- | -------------------------------------- | ---------------------------------- |
| Q1 结束  | 是否建立产品 + AI 的双重基础认知       | 自测 + 1 份 PRD + 1 套 Prompt 模板 |
| Q2 结束  | 是否具备 AI 应用方案设计与基础构建能力 | 方案评审 + Demo 演示               |
| Q3 结束  | 是否能完成完整项目闭环                 | 项目汇报 + 用户反馈 + 复盘         |
| Q4 结束  | 是否形成企业级上岗材料                 | 作品集 + 面试题库 + 系统/商业分析  |

#### 1.2.1 季度复盘建议问题

1. 这一季度我最能证明能力的材料是什么？
2. 哪个能力是“以为会了”，但实际还不能独立完成？
3. 哪个阶段做了很多输入，却没有形成输出？
4. 如果下季度只能补 2 个短板，应该选什么？

## 九、面试准备详细方案

> AI产品经理面试有其特殊性，以下提供系统化的准备方案。

### 9.1 面试考察维度

| 维度       | 权重 | 典型问题                           |
| ---------- | ---- | ---------------------------------- |
| 产品思维   | 30%  | 如何发现和验证需求？如何设计MVP？  |
| AI技术理解 | 25%  | 解释RAG原理？如何评估模型效果？    |
| 项目经验   | 25%  | 介绍你做过的AI项目？遇到什么挑战？ |
| 系统设计   | 10%  | 设计一个AI客服系统？               |
| 文化匹配   | 10%  | 为什么想做AI产品经理？             |

### 9.2 高频面试题及回答框架

**产品思维类**：

1. **"如何发现用户需求？"**
   - 回答框架：用户调研方法 → 需求分析 → 优先级排序 → 验证方案
   - 加分点：结合AI产品特殊性（如Prompt驱动的需求验证）

2. **"如何设计一个AI产品的MVP？"**
   - 回答框架：核心假设 → 最小验证方案 → 效果指标 → 迭代计划
   - 加分点：考虑AI的不确定性和成本

**AI技术类**：3. **"解释一下RAG的原理"**

- 回答框架：问题背景 → 核心流程 → 关键组件 → 效果评估
- 加分点：从产品视角讲，关注效果和成本而非纯技术

4. **"如何评估AI功能的效果？"**
   - 回答框架：指标体系 → 评测方法 → A/B测试 → 持续优化
   - 加分点：区分离线评测和在线评测

5. **"如何处理模型幻觉？"**
   - 回答框架：幻觉类型 → 检测方法 → 缓解策略 → 兜底方案
   - 加分点：结合具体场景给出产品层面的解决方案

**项目经验类**：6. **"介绍你做过的AI项目"**

- 回答框架（STAR法则）：
  - Situation：项目背景和挑战
  - Task：你的职责和目标
  - Action：你做了什么（重点）
  - Result：效果和数据

7. **"项目中遇到的最大挑战是什么？"**
   - 回答框架：挑战描述 → 分析原因 → 解决方案 → 效果验证 → 经验总结
   - 加分点：展示解决复杂问题的能力

**系统设计类**：8. **"设计一个AI客服系统"**

- 回答框架：
  1.  需求分析（用户是谁、解决什么问题）
  2.  核心功能（意图识别、多轮对话、知识库）
  3.  技术方案（模型选型、RAG、对话管理）
  4.  效果评估（准确率、满意度、成本）
  5.  迭代计划（冷启动、优化路径）

### 9.3 面试准备时间线

| 时间          | 任务                 | 产出            |
| ------------- | -------------------- | --------------- |
| 第10个月      | 整理面试题库         | 30+面试题及回答 |
| 第11个月第1周 | 优化简历和作品集     | 最终版简历      |
| 第11个月第2周 | 模拟面试（产品思维） | 3次模拟面试     |
| 第11个月第3周 | 模拟面试（AI技术）   | 3次模拟面试     |
| 第11个月第4周 | 模拟面试（系统设计） | 2次模拟面试     |
| 第12个月      | 正式面试 + 复盘优化  | Offer           |

## 十、学习效果验证方法

> 如何判断自己真的学懂了？以下提供系统化的验证方法。

### 10.1 概念掌握验证

**自测标准**（适用于每个AI概念）：

- [ ] 能用自己的话解释（不背定义）
- [ ] 能画出概念之间的关系图
- [ ] 能举出3个实际应用例子
- [ ] 能回答一个初学者的提问
- [ ] 能在产品决策中正确运用

**概念自测题示例**：

| 概念             | 自测问题                                                           |
| ---------------- | ------------------------------------------------------------------ |
| LLM              | 向一个不懂技术的人解释大语言模型是什么，以及它能做什么、不能做什么 |
| Token            | 估算"今天天气真好，适合出去散步"这句话大约有多少Token？            |
| RAG              | 为什么企业知识库不用直接让LLM回答，而要用RAG？                     |
| Agent            | Agent和普通聊天机器人有什么区别？什么场景适合用Agent？             |
| Embedding        | 如果两段文本的Embedding向量很接近，说明什么？                      |
| 幻觉             | 你设计了一个AI法律咨询产品，如何降低幻觉风险？                     |
| Prompt           | 设计一个Prompt，让AI帮你写一份产品周报，要求格式统一、内容完整     |
| Function Calling | 什么场景下需要用Function Calling而不是普通对话？                   |

### 10.2 能力等级验证

**每季度自评**：对照能力模型，逐项评估自己的等级

**验证方法**：

| 能力       | 验证方式            | 通过标准                     |
| ---------- | ------------------- | ---------------------------- |
| 需求分析   | 独立完成1次用户调研 | 输出可用的用户画像和需求列表 |
| 产品设计   | 独立编写1份PRD      | PRD可直接用于开发            |
| Prompt设计 | 设计10个Prompt      | 8个以上效果满意              |
| RAG设计    | 设计1个知识库方案   | 方案完整，考虑了关键环节     |
| Agent设计  | 设计1个Agent方案    | 方案可执行，有容错设计       |
| 数据分析   | 分析1组产品数据     | 能发现3个以上数据洞察        |
| 成本估算   | 估算1个AI产品成本   | 误差在30%以内                |

### 10.3 项目成果验证

**项目完成度检查清单**：

- [ ] 有明确的目标用户和问题定义
- [ ] 有完整的需求分析文档
- [ ] 有可运行的产品Demo
- [ ] 有至少5个用户的测试反馈
- [ ] 有数据效果分析（即使数据量小）
- [ ] 有项目复盘和改进计划

**项目质量标准**：

- 基本可用：功能完整，能解决核心问题
- 良好：有用户测试数据和迭代优化
- 优秀：有完整的文档、数据支撑和深度复盘

## 十二、年度追踪仪表盘

建议每月底更新一次以下表格，确保全年目标不失焦。

| 维度     | 追踪问题                            | 本月状态 |
| -------- | ----------------------------------- | -------- |
| 学习进度 | 本月是否完成既定主题？              | ⬜       |
| 产出质量 | 是否沉淀了可复用、可展示的成果？    | ⬜       |
| 项目实践 | 是否有真实项目或 Demo 推进？        | ⬜       |
| 能力提升 | 哪一项能力确实比上月更强？          | ⬜       |
| 求职准备 | 是否新增了能放进作品集/简历的材料？ | ⬜       |
| 风险预警 | 哪个目标明显延期或质量不达标？      | ⬜       |

### 12.1 适合写在月末复盘里的 5 个问题

1. 这个月最有价值的输出是什么？
2. 哪个阶段投入很多但回报不高？
3. 哪个知识点已经能用于真实产品决策？
4. 哪个项目还缺少数据、评测或复盘支撑？
5. 下个月最重要的一件事是什么？

---

> 📌 **关键提醒**：计划是死的，人是活的。根据实际情况灵活调整，但核心目标不要轻易改变。坚持12个月，你会看到巨大的成长。
