## 一、RAG基础

### 1.1 核心概念

**Q1: 什么是RAG？为什么需要RAG？**

```
参考答案要点：
- 定义：Retrieval-Augmented Generation，检索增强生成
- 原理：先检索相关文档，再基于文档生成回答
- 需要原因：
  1. 知识更新：模型知识有截止日期
  2. 私有知识：模型未训练企业私有数据
  3. 可溯源：回答可追溯到来源
  4. 减少幻觉：基于事实生成
```

**Q2: RAG和微调（Fine-tuning）的区别？**

```
参考答案要点：
- RAG：
  - 不改变模型参数
  - 动态更新知识（换文档即可）
  - 适合频繁更新的知识
  - 成本低

- 微调：
  - 改变模型参数
  - 知识固化在模型中
  - 适合固定领域知识
  - 成本高

- 选择：
  - 知识频繁更新 → RAG
  - 需要改变模型行为 → 微调
  - 两者可结合使用
```

**Q3: 请描述RAG的完整流程**

```
参考答案要点：
1. 文档处理
   - 文档解析（PDF/Word/HTML）
   - 文本清洗（去噪、去重）
   - 文本分块（Chunking）

2. 索引构建
   - Embedding生成
   - 向量数据库存储
   - 索引优化（HNSW/IVF）

3. 检索阶段
   - Query向量化
   - 相似度检索（TopK）
   - 重排序（Rerank）

4. 生成阶段
   - Prompt构建（Query + Context）
   - 大模型生成
   - 后处理（引用标注）
```

### 1.2 技术细节

**Q4: 文本分块（Chunking）有哪些策略？**

```
参考答案要点：
- 固定长度分块：
  - 按字符/Token数切分
  - 优点：简单、均匀
  - 缺点：可能切断语义

- 语义分块：
  - 按段落/句子/主题边界
  - 优点：语义完整
  - 缺点：块大小不均

- 递归分块：
  - 层级切分（章→节→段）
  - 优点：保留结构
  - 适用：结构化文档

- 重叠策略：
  - 相邻块有重叠
  - 避免上下文断裂
```

**Q5: 什么是Embedding？常见的Embedding模型有哪些？**

```
参考答案要点：
- 定义：将文本映射为高维向量，语义相似则向量相近

- 常见模型：
  - OpenAI：text-embedding-3-large（3072维）
  - BGE：BGE-large-zh（1024维，中文好）
  - M3E：m3e-base（768维，轻量）
  - GTE：gte-large（通用）

- 选择因素：
  - 语言支持
  - 维度大小
  - 效果 vs 成本
```

## 二、向量数据库

### 2.1 选型对比

**Q6: 常见的向量数据库有哪些？如何选择？**

```
参考答案要点：
- Milvus：
  - 开源、分布式
  - 适合大规模数据
  - 功能丰富

- Pinecone：
  - 全托管SaaS
  - 简单易用
  - 成本较高

- Weaviate：
  - 开源、GraphQL接口
  - 支持混合搜索
  - 模块化设计

- Qdrant：
  - 开源、Rust实现
  - 高性能
  - 过滤搜索强

- 选择因素：
  - 数据规模
  - 部署方式
  - 性能要求
  - 预算
```

**Q7: 向量检索的索引算法有哪些？**

```
参考答案要点：
- HNSW（Hierarchical Navigable Small World）：
  - 图索引
  - 速度快、精度高
  - 内存占用大

- IVF（Inverted File Index）：
  - 聚类+倒排
  - 内存省、速度中等
  - 需要调参

- FLAT：
  - 暴力搜索
  - 精度最高
  - 速度慢、只适合小数据

- PQ（Product Quantization）：
  - 向量量化压缩
  - 节省内存
  - 精度有损失
```

### 2.2 性能优化

**Q8: 如何提升RAG的检索效果？**

```
参考答案要点：
- 数据层面：
  - 文档清洗：去噪、去重
  - 分块优化：合适的大小和重叠
  - 元数据：添加标题、标签等

- 检索层面：
  - 混合检索：向量+关键词
  - 重排序：精排模型
  - Query扩展：同义词、相关词

- 生成层面：
  - Prompt优化：明确指令
  - 上下文压缩：去除冗余
  - 引用标注：增强可信度
```

**Q9: 如何处理长文档的RAG？**

```
参考答案要点：
- 文档切分：
  - 层级切分（章→节→段）
  - 保持上下文关联

- 摘要索引：
  - 文档级摘要
  - 两级检索（先文档后段落）

- 递归检索：
  - 先检索相关章节
  - 再检索章节内段落

- 长上下文模型：
  - 使用支持长上下文的模型
  - 一次性输入更多内容
```

## 三、评估优化

### 3.1 评估指标

**Q10: 如何评估RAG系统的效果？**

```
参考答案要点：
- 检索指标：
  - 召回率：相关文档被检索到的比例
  - 精确率：检索结果中相关文档比例
  - MRR：平均倒数排名
  - NDCG：归一化折损累积增益

- 生成指标：
  - 忠实度：回答与文档一致性
  - 相关性：回答与问题相关度
  - 流畅度：语言自然度
  - 引用准确率：引用来源正确性

- 业务指标：
  - 用户满意度
  - 任务完成率
  - 人工介入率
```

**Q11: 什么是RAG的" Lost in the Middle "问题？**

```
参考答案要点：
- 现象：模型对上下文中间部分的信息提取能力弱
- 原因：注意力机制对首尾关注更多
- 解决：
  - 重排序：将重要信息放首尾
  - 压缩：减少上下文长度
  - 分步：多次检索生成
```

### 3.2 优化策略

**Q12: RAG有哪些高级优化技术？**

```
参考答案要点：
- Query优化：
  - Query重写：扩展、澄清
  - Query分解：复杂问题拆分为子问题
  - HyDE：假设文档嵌入

- 检索优化：
  - 多路召回：向量+关键词+图谱
  - 重排序：Cross-Encoder精排
  - 反馈学习：基于点击反馈优化

- 生成优化：
  - 上下文压缩：去除冗余
  - 多文档融合：整合多个来源
  - 引用生成：自动标注来源
```

## 四、工程落地

### 4.1 架构设计

**Q13: RAG系统的工程架构如何设计？**

```
参考答案要点：
- 数据流：
  - 文档摄入 → 解析 → 分块 → Embedding → 存储
  - Query → Embedding → 检索 → 重排 → 生成

- 组件：
  - 文档处理器
  - Embedding服务
  - 向量数据库
  - 检索服务
  - 生成服务

- 扩展性：
  - 异步处理文档
  - 缓存热点查询
  - 多模型路由
```

**Q14: RAG系统的成本如何优化？**

```
参考答案要点：
- 存储成本：
  - 向量量化压缩
  - 分层存储（热/温/冷）
  - 定期清理过期数据

- 计算成本：
  - 缓存Embedding结果
  - 批量处理请求
  - 选择性价比高的Embedding模型

- 生成成本：
  - 缓存常见回答
  - 使用小模型处理简单查询
  - Prompt压缩
```

### 4.2 常见问题

**Q15: RAG系统上线后效果不佳，如何排查？**

```
参考答案要点：
- 检索问题：
  - 检查文档质量
  - 测试Embedding相似度
  - 调整TopK数量

- 生成问题：
  - 检查Prompt设计
  - 测试模型能力
  - 检查上下文长度

- 数据问题：
  - 文档是否覆盖用户问题
  - 文档是否过时
  - 分块是否合理

- 系统问题：
  - 日志分析
  - 性能监控
  - 用户反馈收集
```
