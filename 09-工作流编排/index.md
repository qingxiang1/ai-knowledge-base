## 关键术语速查

| 术语 | 说明 |
|------|------|
| **Workflow** | 工作流，将多个 AI 节点按逻辑顺序编排的自动化流程 |
| **Node** | 工作流节点，工作流中的单个处理单元（LLM 调用、工具调用、条件判断） |
| **Chain** | LangChain 的链式调用，按顺序执行多个 LLM 调用 |
| **Agent** | LangChain 的智能代理，自主决策调用工具和 LLM |
| **Graph** | LangGraph 的状态机图，定义节点、边、状态的工作流 |
| **State** | LangGraph 的状态，工作流执行过程中传递的数据 |
| **Edge** | LangGraph 的边，定义节点间的连接和条件分支 |
| **Trigger** | n8n 的触发器，启动工作流的条件（定时、API、事件） |
| **Condition** | 工作流的条件节点，根据条件选择不同分支 |
| **Loop** | 工作流的循环节点，重复执行某个流程 |
| **Parallel** | 工作流的并行节点，同时执行多个流程 |
| **Error Handling** | 工作流的错误处理，节点失败时的降级策略 |
| **Retry** | 工作流的重试机制，节点失败后自动重试 |

## 学习建议

1. **先上手低代码平台**：Dify/Coze 是最快的入门方式，先建立工作流的基本认知
2. **理解设计方法论**：工作流设计方法论是核心，掌握节点类型、流程模式、错误处理
3. **重点突破 LangGraph**：LangGraph 是 Agent 工作流的标准实现，是进阶必学
4. **动手实践**：用 Dify/Coze 搭建一个完整 AI 应用，理解工作流全流程
5. **关注企业场景**：企业自动化需要考虑集成、监控、运维，n8n 是最佳学习素材
6. **平台选型思维**：不同场景选择不同平台，低代码 vs 代码框架各有优劣
7. **性能与成本**：工作流的节点数量直接影响成本和响应时间，需要优化
