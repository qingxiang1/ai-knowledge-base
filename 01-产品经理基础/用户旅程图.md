# 用户旅程图

> 描述用户与产品交互的全过程，发现体验痛点与机会点

---

## 一、用户旅程图概述

### 1.1 什么是用户旅程图

**用户旅程图（Customer Journey Map）** 是以用户视角描述其与产品/服务交互全过程的可视化工具，展示用户在每个阶段的**行为、触点、想法和情绪**。

### 1.2 用户旅程图的价值

```
用户旅程图解决的问题：

用户角度           产品角度
   │                  │
   ▼                  ▼
"我遇到什么问题" → "我们能提供什么"
"我怎么发现的"   → "触点覆盖了吗"
"我为什么离开"   → "哪里需要优化"
"我还想要什么"   → "还有哪些机会"
```

| 价值点 | 说明 |
|--------|------|
| 全局视角 | 从用户视角看问题，而非从功能角度 |
| 发现断点 | 识别用户流程中的问题和机会 |
| 跨部门协作 | 让所有团队对齐用户视角 |
| 优先级决策 | 基于用户体验而非主观判断排序需求 |

### 1.3 与其他工具的区别

| 工具 | 关注点 | 视角 |
|------|--------|------|
| 用户画像 | 用户是谁 | 静态的「人」 |
| 用户旅程图 | 用户做了什么 | 动态的「过程」 |
| 流程图 | 内部流程 | 企业视角 |
| MindMap | 功能结构 | 产品视角 |

## 二、用户旅程图的核心要素

### 2.1 标准要素

```
用户旅程图要素：

├── 用户画像（Who）
│   └── 明确旅程是为哪个用户类型绘制
│
├── 场景/目标（Why）
│   └── 用户想达成什么目标
│
├── 阶段划分（When）
│   └── 用户经历哪几个主要阶段
│
├── 触点（Touchpoints）
│   └── 用户在哪些地方与产品互动
│
├── 行为（Actions）
│   └── 用户具体做了什么
│
├── 想法（Thoughts）
│   └── 用户当时在想什么
│
├── 情绪（Emotions）
│   └── 用户感受如何
│
├── 痛点（Pain Points）
│   └── 用户的困扰和障碍
│
└── 机会点（Opportunities）
    └── 可改进或创新的地方
```

### 2.2 AI产品特殊要素

| 特殊要素 | 说明 | 示例 |
|---------|------|------|
| AI交互节点 | 哪些环节涉及AI | AI生成、AI推荐、AI对话 |
| 信任曲线 | 用户对AI的信任变化 | 从怀疑到依赖 |
| 兜底触发 | 何时需要人工介入 | AI无法回答时 |
| 反馈行为 | 用户如何纠正AI | 点赞/点踩/修改 |

## 三、用户旅程图构建流程

### 3.1 整体流程

```
确定范围 → 收集数据 → 定义阶段 → 填充内容 → 识别痛点 → 提炼机会 → 验证迭代
    │                                                        │
    └────────────────────────────────────────────────────────┘
```

### 3.2 Step 1：确定范围

**关键问题**：
- 这个旅程图覆盖什么场景？
- 针对哪个用户画像？
- 旅程的起点和终点是什么？

**示例**：

```markdown
## 旅程图范围定义

### 场景
用户首次使用AI客服解决问题的完整流程

### 目标用户
效率优先的内容创作者（小美）

### 起点终点
起点：用户进入客服页面
终点：问题解决或转人工

### 假设前提
- 用户已有账号
- 问题是常见问题类型
```

### 3.3 Step 2：收集数据

#### 数据来源

| 数据类型 | 来源 | 价值 |
|---------|------|------|
| 用户访谈 | 深度访谈用户 | 了解想法和情绪 |
| 问卷调研 | 大规模用户 | 验证普遍性 |
| 行为数据 | 产品埋点 | 了解实际行为 |
| 客服记录 | 客服工单 | 发现常见问题 |
| 眼动测试 | 可用性测试 | 发现注意力分布 |

#### AI产品特别关注

- 用户如何发现AI能力
- 初次使用时的心理障碍
- 信任建立的转折点
- 用户如何判断AI回答质量

### 3.4 Step 3：定义阶段

#### 通用阶段框架

```
AIDA模型（适用于营销转化）：
Attention（注意）→ Interest（兴趣）→ Desire（欲望）→ Action（行动）

AISAS模型（适用于互联网产品）：
Attention（注意）→ Interest（兴趣）→ Search（搜索）→ Action（行动）→ Share（分享）

用户体验五阶段模型：
触达 → 使用 → 适应 → 依赖 → 推荐
```

#### AI产品阶段建议

| 产品类型 | 建议阶段 |
|---------|---------|
| AI对话产品 | 发起对话 → 等待响应 → 接收回答 → 采纳/修改 → 结束/追问 |
| AI写作产品 | 明确需求 → 输入信息 → 生成内容 → 修改调整 → 保存/发布 |
| AI客服产品 | 发起咨询 → 问题识别 → AI处理 → 满意度确认 → 问题解决 |
| AI搜索产品 | 输入查询 → 结果呈现 → 结果浏览 → 结果使用 → 反馈评价 |

## 五、用户旅程图的呈现形式

### 5.1 表格形式

适合详细记录和文档化：

```markdown
| 阶段 | 行为 | 触点 | 想法 | 情绪 | 痛点 | 机会点 |
|------|------|------|------|------|------|--------|
| 构思期 | 刷热点榜单 | 微博/小红书 | 有热点可以追 | 兴奋 | 判断哪个热点值得写 | AI热点推荐 |
| 初稿期 | 收集资料 | 竞品文章 | 资料太多太乱 | 焦虑 | 整理耗时 | 自动整理摘要 |
| 生成期 | 输入需求 | AI助手界面 | AI能写出什么 | 期待又担心 | AI味道太重 | 风格学习 |
| 修改期 | 调整内容 | 文档编辑器 | 改起来麻烦 | 不耐烦 | 改动量大 | 智能优化 |
```

## 六、从旅程图到行动

### 6.1 痛点→机会点转化

| 痛点类型 | 转化方法 | AI产品机会 |
|---------|---------|-----------|
| 流程断点 | 补充触点 | AI预判用户需求 |
| 情绪低谷 | 优化体验 | AI主动安抚/解释 |
| 认知障碍 | 降低门槛 | AI主动引导 |
| 效率低下 | 自动化 | AI替代/辅助 |
| 信任缺失 | 透明化 | 显示AI思考过程 |

### 6.2 优先级排序

```
机会点优先级矩阵：

              高影响力
                │
    快速胜利      │      战略投入
    （小投入      │       （大投入
     大回报）      │        大回报）
                │
   ─────────────┼─────────────
                │
    降低优先级    │       重新设计
    （可后续      │      （高投入
     再做）       │       低回报）
                │
              低影响力
                 │
           低成本         高成本
              ←─────────────→
                   投入成本
```

### 6.3 输出物

```markdown
## 用户旅程优化方案

### 基于旅程图识别的TOP3机会点

**机会点1：AI热点推荐**
- 来源痛点：构思期，用户难以判断热点价值
- 解决方案：基于历史数据训练热点预测模型
- 预期收益：选题效率提升50%
- 实施难度：中
- 优先级：P1

**机会点2：风格学习**
- 来源痛点：修改期，AI生成内容风格不匹配
- 解决方案：学习用户历史文章风格
- 预期收益：修改率降低40%
- 实施难度：高
- 优先级：P2

**机会点3：实时反馈**
- 来源痛点：生成期，用户不知道AI在做什么
- 解决方案：显示AI思考过程和置信度
- 预期收益：信任度提升
- 实施难度：低
- 优先级：P1
```
