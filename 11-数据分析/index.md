## 关键术语速查

| 术语 | 说明 |
|------|------|
| **数据分析** | 通过数据采集、处理、分析，发现规律、支持决策的方法 |
| **数据埋点** | 在产品中埋入数据采集代码，记录用户行为事件 |
| **事件** | 用户在产品中的行为，如点击、搜索、对话、提交 |
| **属性** | 事件的详细信息，如搜索关键词、对话内容、提交时间 |
| **Token 成本** | AI 产品的核心成本指标，每次对话消耗的 Token 数量 |
| **响应时间** | AI 产品的核心性能指标，从请求到响应的时间 |
| **准确率** | AI 产品的核心质量指标，AI 输出的正确率 |
| **用户满意度** | AI 产品的核心体验指标，用户对 AI 输出的满意度评分 |
| **留存率** | AI 产品的核心增长指标，用户持续使用产品的比例 |
| **A/B 测试** | 通过对比两个版本的效果，验证产品优化方案的方法 |
| **显著性检验** | A/B 测试的核心方法，判断两个版本差异是否显著 |
| **漏斗分析** | 用户行为分析方法，分析用户完成某个流程的转化率 |
| **留存分析** | 用户行为分析方法，分析用户在不同时间段的留存情况 |
| **路径分析** | 用户行为分析方法，分析用户在产品中的行为路径 |
| **用户分群** | 用户行为分析方法，根据用户行为特征将用户分群 |
| **数据可视化** | 通过图表、Dashboard 等方式呈现数据分析结果 |
| **数据驱动决策** | 基于数据分析结果进行产品决策的方法 |

## 学习建议

1. **先理解基础**：数据分析基础是核心，理解数据分析概念、方法论、流程
2. **掌握 AI 产品指标**：AI 产品有特有的数据指标（Token 成本、响应时间、准确率），是必学内容
3. **重点突破 Prompt 评估**：Prompt 效果评估是 AI 产品优化的核心方法
4. **动手实践**：定义一个 AI 产品的核心指标，设计数据采集方案
5. **关注 A/B 测试**：A/B 测试是验证产品优化效果的标准方法
6. **建立数据文化**：数据驱动决策需要建立数据监控、数据文化，是长期工作
7. **可视化思维**：数据可视化是呈现分析结果的关键，需要掌握 Dashboard 设计
