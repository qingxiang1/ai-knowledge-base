# Project01 企业级写作工作流

## 功能目标

在现有 AI 写作助手基础上，实现一条可直接扩展到企业场景的最小闭环：作者生成草稿，提交审核，审核人通过或驳回，发布人执行内部模拟发布，并保留完整审计轨迹。

## 本次范围

- 企业模板选择
- 草稿生成与重新生成
- 单据状态流转
- 作者、审核人、发布人三类角色模拟
- 内部模拟发布
- 审计日志与单据列表
- 本地 JSON 持久化，支持服务重启后保留单据和审计记录
- 品牌规则校验与提审阻断，支持识别阻断项与提示项

## 不在本次范围

- 真实登录鉴权
- 数据库持久化
- 外部 CMS / 邮件系统真实联动
- 多租户、组织架构、权限中心

## 验收标准

### 正常流程

- AC-001
  Given 当前用户以作者角色进入写作工作台且已填写标题、模板和业务摘要
  When 点击“生成企业草稿”
  Then 系统创建一条状态为 `draft` 的单据，并返回可查看的企业草稿内容

- AC-002
  Given 当前存在一条 `draft` 或 `rejected` 状态的单据
  When 作者点击“提交审核”
  Then 单据状态变更为 `in_review`，并记录一次“提交审核”的审计日志

- AC-003
  Given 当前用户以审核人角色查看一条 `in_review` 状态的单据
  When 点击“审核通过”
  Then 单据状态变更为 `approved`，并记录审核意见和审计日志

- AC-004
  Given 当前用户以发布人角色查看一条 `approved` 状态的单据
  When 点击“内部模拟发布”
  Then 单据状态变更为 `published`，并记录发布备注和审计日志

### 边界与异常

- AC-005
  Given 标题、模板或业务摘要为空
  When 作者尝试生成草稿
  Then 系统阻止提交并提示具体缺失项

- AC-006
  Given 单据已进入 `in_review`、`approved` 或 `published` 状态
  When 作者尝试直接修改并重新生成当前单据
  Then 系统阻止编辑并提示需要新建草稿

- AC-007
  Given 当前用户不是对应流程角色
  When 尝试执行不属于自己角色的动作
  Then 系统阻止操作并提示当前角色无权限

- AC-008
  Given 审核人准备驳回一条 `in_review` 单据
  When 未填写驳回原因直接点击“驳回修改”
  Then 系统阻止驳回并提示必须填写驳回原因

### 业务规则

- AC-009
  Given 一条单据进入任意流程动作
  When 系统完成状态变更
  Then 必须记录动作名称、前后状态、操作者角色、操作者姓名、备注和时间

- AC-010
  Given 单据被重新生成草稿
  When 生成成功
  Then 单据版本号递增，更新时间刷新，审计轨迹新增一条记录

- AC-011
  Given 作者生成或重新生成一条企业草稿
  When 系统完成内容生成
  Then 系统必须自动执行一次品牌规则校验，并保存校验时间、校验状态和命中问题列表

- AC-012
  Given 一条 `draft` 或 `rejected` 状态的单据存在阻断级品牌规则问题
  When 作者尝试提交审核
  Then 系统阻止提审并提示阻断原因，单据状态保持不变

- AC-013
  Given 当前工作台已选中某条企业单据
  When 单据存在品牌规则命中结果
  Then 前端必须展示合规状态、命中问题、命中文本和修正建议，帮助作者完成修改
