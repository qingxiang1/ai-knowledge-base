## 关键术语速查

| 术语 | 说明 |
|------|------|
| **系统设计** | 设计 AI 产品的系统架构，包括组件设计、接口设计、数据流设计 |
| **系统架构** | AI 产品的系统架构，包括前端、后端、AI 服务、数据存储 |
| **需求分析** | 分析系统的功能需求、性能需求、安全需求 |
| **架构设计** | 设计系统的组件架构、接口架构、数据流架构 |
| **技术选型** | 选择系统的技术栈，包括模型、框架、平台、工具 |
| **性能优化** | 优化系统的响应时间、吞吐量、成本 |
| **响应时间优化** | 优化系统的响应时间，如缓存、预加载、并发处理 |
| **吞吐量优化** | 优化系统的吞吐量，如负载均衡、分布式处理 |
| **成本优化** | 优化系统的成本，如 Token 成本、存储成本、运维成本 |
| **监控优化** | 优化系统的监控，如性能监控、错误监控、业务监控 |
| **安全设计** | 设计系统的安全机制，如数据安全、隐私保护、访问控制 |
| **数据安全** | 保护系统数据的安全，如加密、备份、访问控制 |
| **隐私保护** | 保护用户隐私，如数据脱敏、隐私合规 |
| **访问控制** | 控制系统的访问权限，如身份认证、权限管理 |
| **安全审计** | 审计系统的安全事件，如日志审计、安全报告 |
| **RAG 系统** | RAG 系统架构，包括数据管道、向量检索、重排序、缓存 |
| **Agent 系统** | Agent 系统架构，包括任务规划、记忆系统、工具集成、反思 |
| **MCP 平台** | MCP 平台架构，包括服务注册、能力发现、调用管理、监控 |
| **工作流平台** | 工作流平台架构，包括节点设计、流程编排、状态管理、错误处理 |

## 学习建议

1. **先理解方法**：系统设计方法论是核心，理解需求分析、架构设计、技术选型、性能优化、安全设计
2. **重点突破核心系统**：RAG系统/Agent系统是当前主流 AI 产品系统，是必学内容
3. **动手实践**：分析一个 AI 产品的系统架构，理解架构设计
4. **关注性能优化**：系统性能优化是关键，需要优化响应时间、吞吐量、成本
5. **设计安全**：系统安全设计是必要，需要设计数据安全、隐私保护、访问控制
6. **平台思维**：MCP平台/工作流平台是进阶内容，需要理解平台级架构设计
7. **架构演进**：系统架构需要持续演进，建立架构演进机制
