# Prompt优化技巧

> 掌握系统化的Prompt优化方法，持续提升AI输出质量

---

## 一、Prompt优化概述

### 1.1 为什么需要持续优化

```
Prompt优化的本质：

初版Prompt ──→ 测试输出 ──→ 发现问题 ──→ 优化Prompt ──→ 再测试
      ↑                                          │
      └──────────────────────────────────────────┘

优化循环：
1. 定义成功标准（什么算"好"的输出？）
2. 收集测试用例（覆盖各种场景）
3. 运行测试，记录输出
4. 对比成功标准，找出差距
5. 调整Prompt，消除差距
6. 重复步骤3-5直到满足标准
```

### 1.2 优化目标

| 目标 | 说明 | 衡量方式 |
|------|------|---------|
| **准确性** | 输出内容正确 | 与标准答案对比 |
| **相关性** | 输出与问题相关 | 人工判断 |
| **完整性** | 不遗漏关键信息 | 检查清单核对 |
| **格式合规** | 符合格式要求 | 自动校验 |
| **一致性** | 相似输入得到相似输出 | 变异测试 |
| **安全性** | 不输出有害内容 | 安全测试集 |

## 二、基础优化技巧

### 2.1 指令优化

```
指令优化的DO和DON'T：

✅ DO：
- 使用动词开头："分析..." "列出..." "比较..."
- 具体明确："列出5个策略" 而非 "说说策略"
- 分解复杂任务："第一步...第二步...第三步..."
- 说明优先级："最重要的是..." "优先考虑..."

❌ DON'T：
- 模糊表达："写个好点的" "分析一下"
- 双重否定："不要不包含"
- 矛盾指令："详细但简短"
- 开放式问题："关于这个你怎么看"
```

### 2.2 上下文优化

```
上下文提供的技巧：

1. 关键信息前置
   ✅ "背景：我们是B2B SaaS公司（关键信息）
        成立3年，ARR 1000万（次要信息）
        主要客户是中小企业（次要信息）"

2. 使用分隔符
   ### 背景 ###
   [背景信息]
   ### 任务 ###
   [任务指令]

3. 去除无关信息
   ❌ 包含大量与任务无关的背景
   ✅ 只提供影响输出的关键上下文

4. 结构化上下文
   背景：
   - 公司阶段：A轮
   - 产品类型：SaaS
   - 目标用户：中小企业
```

### 2.3 输出控制优化

```
输出控制技巧：

1. 格式模板
   "请按以下格式输出：
   结论：[一句话结论]
   理由：
   1. [理由1]
   2. [理由2]"

2. 长度控制
   "字数控制在200字以内"
   "不超过3个要点"
   "用一句话总结"

3. 风格控制
   "用产品经理的语言"
   "像给CEO汇报一样简洁"
   "用通俗的比喻解释"

4. 排除控制
   "不要提及竞争对手"
   "避免使用专业术语"
   "不要给出医疗建议"
```

## 三、进阶优化技巧

### 3.1 迭代优化法

```
迭代优化流程：

版本1：基础Prompt
"分析这个产品的优缺点"
→ 输出泛泛，缺乏深度

版本2：增加角色
"你是一位资深产品经理，分析这个产品的优缺点"
→ 有专业视角，但仍不够具体

版本3：增加维度
"你是一位资深产品经理，从用户体验、商业价值、技术实现
 三个维度分析这个产品的优缺点，每个维度给出3个要点"
→ 结构清晰，但有遗漏

版本4：增加约束
"...（版本3内容）
 要求：
  - 优点和缺点各不超过3个
  - 每个要点附带具体例子
  - 最后给出总体评价和建议"
→ 输出质量显著提升

版本5：增加示例
"...（版本4内容）
 示例格式：
  优点1：[要点] - [例子] - [影响]"
→ 格式统一，可直接使用
```

### 3.2 A/B测试法

```
Prompt A/B测试：

场景：优化商品描述生成Prompt

版本A：
"为以下商品写一段吸引人的描述：
 [商品信息]"

版本B：
"你是一位电商文案专家，为以下商品写描述：
 [商品信息]
 要求：
 - 突出3个核心卖点
 - 使用感官化语言
 - 控制在100字以内"

测试方法：
1. 准备20个不同商品
2. 两个版本各生成20个描述
3. 让目标用户（或评估模型）打分
4. 统计哪个版本平均分更高
5. 分析高分的共同特征，融入Prompt
```

### 3.3 问题诊断法

```
输出问题诊断与修复：

问题1：输出太长
诊断：缺少长度约束
修复：添加"控制在300字以内"

问题2：格式不统一
诊断：没有格式示例
修复：提供输入输出示例

问题3：遗漏关键信息
诊断：Prompt没有明确要求
修复：添加"必须包含以下要点..."

问题4：输出有偏见
诊断：Prompt引导了特定方向
修复：使用中性表达，添加"客观分析"

问题5：拒绝回答
诊断：触及安全边界
修复：调整措辞，明确合法场景
```

## 四、场景化优化策略

### 4.1 内容生成类

```
优化策略：

1. 明确受众
   "写给[目标受众]，语言风格[正式/轻松/专业]"

2. 提供参考
   "参考以下风格：[示例文本]"

3. 结构引导
   "按以下结构：
    开头：吸引注意
    中间：核心内容
    结尾：行动号召"

4. 关键词植入
   "必须包含以下关键词：AI、效率、未来"
```

### 4.2 分析类

```
优化策略：

1. 分析框架
   "使用SWOT框架分析"
   "从5W1H角度分析"

2. 维度明确
   "从成本、效果、风险三个维度评估"

3. 深度要求
   "不仅说明是什么，还要分析为什么"
   "给出数据支撑的具体例子"

4. 结论导向
   "最后给出明确的结论和建议"
```

### 4.3 代码类

```
优化策略：

1. 环境说明
   "使用Python 3.9，兼容pandas 1.5"

2. 约束明确
   "时间复杂度不超过O(n)"
   "不使用第三方库"

3. 注释要求
   "关键步骤添加中文注释"
   "函数添加docstring"

4. 测试用例
   "包含以下测试用例：
    输入：[1,2,3] → 输出：6"
```

## 五、常见陷阱与避免

### 5.1 Prompt陷阱

| 陷阱 | 表现 | 避免方法 |
|------|------|---------|
| **过度优化** | Prompt比输出还长 | 保持简洁，聚焦关键约束 |
| **示例污染** | 示例包含错误信息 | 仔细审核示例质量 |
| **角色冲突** | 同时设定矛盾角色 | 明确主次角色 |
| **暗示偏见** | Prompt措辞引导特定答案 | 使用中性语言 |
| **假设前提** | Prompt包含未验证假设 | 区分事实和假设 |

### 5.2 优化检查清单

```
Prompt发布前检查：

□ 指令是否清晰无歧义？
□ 角色设定是否明确？
□ 输出格式是否定义清楚？
□ 约束条件是否合理且不矛盾？
□ 示例是否正确且具代表性？
□ 是否考虑了边界情况？
□ 是否去除了敏感信息？
□ Token长度是否在限制内？
```

Prompt优化

**关键认知**：
1. Prompt优化是一个持续迭代的过程，不是一次性工作
2. 优化前应定义清晰的成功标准和测试集
3. 从简单到复杂逐步优化，避免过度设计
4. A/B测试是验证优化效果的最佳方法
5. AI产品经理应建立Prompt版本管理和效果追踪机制
