# PMF验证

> AI产品市场匹配（Product-Market Fit）验证方法论，帮助创业者科学判断产品是否满足市场需求。

---

## 一、PMF基础

### 1.1 什么是PMF

```
PMF定义：

产品市场匹配 = 产品解决的需求 × 愿意付费的用户群体

当产品满足以下条件时，认为达到PMF：
1. 用户主动推荐（NPS > 40）
2. 留存率健康（次日留存 > 40%，7日留存 > 20%）
3. 付费转化稳定（付费率 > 5%）
4. 获客成本 < 用户生命周期价值（CAC < LTV）
5. 用户流失率可控（月流失 < 5%）
```

### 1.2 AI产品PMF特殊性

```
AI产品PMF的特殊挑战：

1. 效果不确定性
   - 模型输出不稳定
   - 用户期望管理难
   - 需要持续调优

2. 成本结构
   - 推理成本高
   - 单位经济模型压力大
   - 需要找到盈亏平衡点

3. 用户教育
   - Prompt Engineering学习成本
   - AI能力边界认知
   - 使用习惯培养

4. 技术迭代
   - 模型快速升级
   - 功能可能被模型原生替代
   - 需要持续跟进技术趋势
```

### 2.3 付费验证

```
付费验证指标：

1. 转化漏斗
   访问 → 注册 → 激活 → 付费 → 续费
    │      │      │      │      │
   100%   30%    50%    10%    60%

2. 单位经济模型
   - ARPU：每用户平均收入
   - CAC：获客成本
   - LTV：用户生命周期价值
   - 目标：LTV/CAC > 3，回收期 < 12个月

3. AI产品特殊指标
   - 每用户Token消耗
   - 推理成本占比
   - 毛利率 = (收入 - 推理成本) / 收入
   - 目标毛利率 > 60%
```

## 三、验证实验

### 3.1  smoke test（烟雾测试）

```
烟雾测试：在产品开发前验证需求

方法1：落地页测试
- 制作产品介绍页
- 投放广告获取流量
- 收集用户邮箱/预约
- 判断：转化率 > 5% 说明需求存在

方法2：假门测试（Fake Door）
- 在产品中加入"即将上线"功能入口
- 统计点击意愿
- 判断：点击率 > 10% 说明需求强烈

方法3：预售测试
- 产品未开发即开始预售
- 收取定金或全款
- 判断：预售金额 > 目标10% 说明PMF潜力
```

### 3.2  concierge MVP（礼宾式MVP）

```
礼宾式MVP：用人工替代AI，验证核心价值

示例：AI客服产品
- 产品：承诺AI自动回复
- 实际：人工在后台回复
- 目的：验证客户是否愿意为"自动回复"付费
- 指标：客户满意度、付费意愿

示例：AI写作助手
- 产品：承诺AI生成文章
- 实际：人工撰写后伪装成AI生成
- 目的：验证用户对"AI内容"的接受度
- 指标：内容使用率、修改率

关键原则：
- 用户不知道背后是人还是AI
- 重点验证价值，非技术可行性
- 收集反馈指导产品开发
```

## 四、用户研究

### 4.1 用户访谈框架

```
PMF用户访谈问题清单：

背景了解（5分钟）
1. 请介绍一下您的工作/角色？
2. 您目前如何处理[问题领域]？
3. 在这个过程中最大的痛点是什么？

产品使用（10分钟）
4. 您是如何发现我们产品的？
5. 第一次使用时有什么感受？
6. 您最常用的功能是什么？为什么？
7. 有没有遇到过问题或困惑？

价值评估（10分钟）
8. 如果明天无法使用这个产品，您会怎么办？
9. 您会向同事/朋友推荐这个产品吗？为什么？
10. 您愿意为这个产品付费吗？什么价格？

改进建议（5分钟）
11. 如果只能改进一个功能，您会选择什么？
12. 还有什么想告诉我们的吗？
```

## 五、数据指标

### 5.1 AI产品核心指标

| 指标类别 | 具体指标 | 计算公式 | 健康值 |
|:---|:---|:---|:---|
| 获客 | 新增用户 | 日/周/月新增 | 环比增长 > 10% |
| 获客 | CAC | 营销费用 / 新增用户 | < LTV/3 |
| 激活 | 激活率 | 完成核心行为 / 注册数 | > 50% |
| 留存 | 次日留存 | 次日活跃 / 首日新增 | > 30% |
| 留存 | 7日留存 | 7日活跃 / 首日新增 | > 15% |
| 留存 | 月流失率 | 流失用户 / 月初用户 | < 5% |
| 收入 | ARPU | 总收入 / 总用户 | 逐月提升 |
| 收入 | 付费率 | 付费用户 / 总用户 | > 3% |
| 收入 | LTV | ARPU × 用户平均生命周期 | > 3×CAC |
| 成本 | 推理成本占比 | 推理成本 / 总收入 | < 40% |
| 成本 | 毛利率 | (收入-成本) / 收入 | > 60% |
| 推荐 | NPS | 推荐者% - 贬损者% | > 40 |

## 六、决策框架

### 6.1 PMF判断决策树

```
PMF判断决策树：

开始
  │
  ├─ 用户是否主动推荐？
  │   ├─ 是 → NPS > 40？
  │   │       ├─ 是 → 留存是否健康？
  │   │       │       ├─ 是 → 付费转化是否稳定？
  │   │       │       │       ├─ 是 → ✅ 达到PMF，准备规模化
  │   │       │       │       └─ 否 → 优化商业模式
  │   │       │       └─ 否 → 优化产品价值
  │   │       └─ 否 → 提升用户体验
  │   └─ 否 → 产品价值不足，需重新定位
  │
  └─ 是否有付费用户？
      ├─ 是 → 单位经济模型是否健康？
      │       ├─ 是 → 扩大获客
      │       └─ 否 → 降低成本或提价
      └─ 否 → 先验证付费意愿
```

### 6.2 未达到PMF的应对

```
未达到PMF的应对策略：

1. 问题：用户不活跃
   - 诊断：onboarding流程、核心功能价值
   - 行动：简化首次使用、增加引导、优化核心功能

2. 问题：留存率低
   - 诊断：产品价值、使用场景、竞品对比
   - 行动：增加使用场景、提升效果、建立习惯

3. 问题：付费率低
   - 诊断：价格、价值感知、付费点设计
   - 行动：调整定价、增加付费功能、优化付费流程

4. 问题：获客成本高
   - 诊断：渠道效率、产品口碑、市场教育
   - 行动：优化渠道、提升口碑、内容营销

5. 问题：AI效果不稳定
   - 诊断：模型能力、Prompt设计、场景适配
   - 行动：模型调优、工程优化、场景筛选
```
