# 学习路线

> 规划你的 AI 产品成长路径

本模块共 **4** 个章节，帮助你建立系统化的学习规划。

---

## 本章定位

`00-Roadmap` 不是单纯的目录页，而是整个知识库的学习操作系统，主要解决 4 个问题：

1. **知道先学什么**：避免在模型、工具、框架之间盲目跳转
2. **知道为什么学**：明确每个阶段要补齐的能力与岗位价值
3. **知道学到什么程度算过关**：建立阶段性验收标准与产出要求
4. **知道如何转化为求职竞争力**：把学习内容沉淀为方案、项目、作品集和面试话术

### 适合谁使用

| 人群         | 当前状态                 | 本章解决的核心问题                             |
| ------------ | ------------------------ | ---------------------------------------------- |
| 新手入门者   | 对 AI 产品经理了解有限   | 建立职业认知，避免学习顺序混乱                 |
| 技术人员转型 | 懂技术但缺产品方法论     | 快速补齐用户洞察、需求分析、方案表达与商业意识 |
| 传统产品转型 | 懂产品但缺 AI 技术理解   | 建立技术边界认知、模型选型和 AI 方案判断力     |
| 在职提升者   | 已参与 AI 项目但成长无序 | 形成能力矩阵、项目复盘和晋升所需的系统方法     |

## 使用建议

### 第一步：了解全貌（1小时）

1. **先读成长路线**：了解AI产品经理的日常工作、成长阶段和常见误区
2. **对照能力模型**：快速自评，了解自己的能力现状和差距

### 第二步：制定计划（2小时）

1. **参考年度计划**：根据你的背景（技术/产品/零基础），选择对应的差异化方案
2. **落实月度计划**：将年度目标拆解为可执行的月度和周度任务

### 第三步：执行与验证（持续）

1. **按周执行**：每天按计划学习和实践
2. **月度复盘**：每月底对照里程碑检查完成度
3. **自测验证**：用自测题库检验学习效果

### 不同背景的阅读顺序

| 你的背景 | 推荐阅读顺序                                                                | 重点章节                     |

## 企业级学习原则

### 原则一：以岗位产出为导向，而不是以概念记忆为导向

学习每一章时，都应该追问以下问题：

1. 这个知识会影响什么产品决策？
2. 这个概念会转化成什么交付物？
3. 这个能力如何在真实项目里体现价值？

### 原则二：以“可验证成果”替代“学过了”

建议把学习成果固定沉淀为以下 5 类材料：

| 成果类型        | 最低要求                      | 用途                 |
| --------------- | ----------------------------- | -------------------- |
| 概念笔记        | 能用自己的话解释核心概念      | 建立技术理解力       |
| 方案文档        | 至少有 1 份完整产品方案或 PRD | 证明产品设计能力     |
| Prompt/流程资产 | 有结构化模板库和优化记录      | 证明 AI 应用构建能力 |
| 项目案例        | 至少 2 个可讲清楚的项目       | 证明实战能力         |
| 复盘材料        | 有问题、数据、迭代与反思      | 证明成长速度和成熟度 |

### 原则三：用企业级标准要求自己

企业在评估 AI 产品经理时，通常不会只看“懂不懂 AI”，而会看你是否能同时兼顾：

- **用户价值**：是否真的解决了业务问题
- **技术可行性**：方案是否能落地，是否理解能力边界
- **成本效率**：模型、Token、人工运营成本是否可控
- **风险治理**：是否考虑幻觉、安全、合规、数据权限和兜底方案
- **协作推动力**：是否能与研发、算法、设计、运营共同推进交付

## 推荐配套动作

1. **建立一个个人成长档案**：集中存放能力自评、学习计划、项目产出、复盘记录
2. **每月底做一次复盘**：检查是否只是在“看资料”，还是已经形成真实产出
3. **每季度做一次能力重评**：重新对照 `能力模型`，更新下一阶段重点
4. **每完成一个项目就补作品集**：不要等到求职前再整理
5. **持续体验行业产品**：路线规划必须与真实行业变化同步更新

## 关键术语速查

| 术语           | 说明                                                     |
| -------------- | -------------------------------------------------------- |
| **能力模型**   | AI产品经理核心能力框架，包含5大维度、20+子能力           |
| **成长路线**   | 从入门到精通的完整路径，分4个阶段（入门→进阶→高级→专家） |
| **年度计划**   | 12个月系统学习计划，含差异化方案（技术/产品/零基础）     |
| **月度计划**   | 按月细化的学习任务，含详细日计划和自测题库               |
| **差异化方案** | 根据不同背景（技术/产品/零基础）定制的学习路径           |
| **能力缺口**   | 当前能力与目标能力之间的差距，通过自评发现               |
| **里程碑**     | 学习阶段的关键节点，用于检验学习成果                     |
| **自测题库**   | 用于检验学习效果的测试题目集合                           |
| **作品集**     | 学习过程中产出的实践成果集合，用于求职展示               |

## 学习建议

1. **先读成长路线**：建立全局认知，了解 AI 产品经理的日常工作、成长阶段和常见误区
2. **对照能力模型**：快速自评，明确自己的能力现状和差距，制定针对性提升计划
3. **选择差异化方案**：根据你的背景（技术/产品/零基础），选择对应的学习路径
4. **按月执行**：将年度目标拆解为可执行的月度和周度任务，保持持续学习
5. **月度复盘**：每月底对照里程碑检查完成度，及时调整学习节奏
6. **自测验证**：用自测题库检验学习效果，确保知识真正掌握
7. **产出导向**：学习过程中注重实践产出，建立作品集为求职做准备
