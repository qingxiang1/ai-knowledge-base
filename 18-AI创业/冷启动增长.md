# 冷启动增长

> AI产品冷启动与早期增长策略，覆盖获客渠道、增长黑客、社区运营与病毒传播。

---

## 一、冷启动基础

### 1.1 冷启动定义

```
冷启动定义：

产品从0到1获取首批用户的过程，核心挑战：
1. 没有品牌认知
2. 没有用户基础
3. 没有口碑传播
4. 预算有限

AI产品冷启动特殊优势：
- AI本身具有话题性
- 生成内容可传播
- 效果直观易演示
- 技术社区关注度高
```

### 1.2 冷启动阶段划分

```
冷启动阶段：

阶段1：种子用户（0-1000用户）
- 目标：验证产品价值
- 渠道：朋友圈、社群、定向邀请
- 策略：人工服务、深度访谈

阶段2：早期采用者（1000-10000用户）
- 目标：找到增长杠杆
- 渠道：Product Hunt、技术社区、内容营销
- 策略：产品驱动、口碑传播

阶段3：早期大众（10000-100000用户）
- 目标：规模化获客
- 渠道：SEO、付费广告、合作推广
- 策略：数据驱动、渠道优化
```

### 2.3 渠道选择策略

```

│   └── 客单价 < 1万 → PLG + 自助试用

├── C端产品
```

### 5.2 增长模型

```
增长模型：

1. AARRR模型
   Acquisition（获客）→ Activation（激活）→ Retention（留存）→ Revenue（收入）→ Referral（推荐）

2. 北极星指标
   - 定义：最能反映产品核心价值的单一指标
   - AI写作产品：周活跃创作数
   - AI客服产品：周解决问题数
   - AI设计产品：周生成设计数

3. 增长公式
   增长 = 流量 × 转化率 × 留存率 × 付费率 × ARPU × 传播系数
```

### 6.2 失败教训

```
常见失败原因：

1. 过早规模化
   - 产品未验证就大量投放
   - 结果：获客成本高，留存差

2. 忽视产品体验
   - 重营销轻产品
   - 结果：用户来了就走

3. 定位模糊
   - 想服务所有人
   - 结果： nobody cares

4. 忽视数据
   - 凭感觉做决策
   - 结果：资源浪费

5. 增长手段单一
   - 过度依赖某一渠道
   - 结果：渠道变化即崩溃
```
