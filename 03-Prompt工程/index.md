> 提示词设计与优化技巧——AI产品经理的核心实操能力

---

## 本章定位

`03-Prompt工程` 解决的不是“会不会写一句提示词”，而是以下 4 个更关键的问题：

1. **如何把业务目标翻译成可执行的 Prompt 方案**
2. **如何让模型输出更稳定、更可控、更能进入生产系统**
3. **如何把 Prompt 当作产品能力的一部分来设计、评测和版本管理**
4. **如何在企业环境中兼顾效果、成本、安全和协作**

### 学完本章后应达到的最低标准

| 能力       | 达标表现                                                |
| ---------- | ------------------------------------------------------- |
| Prompt设计 | 能写出包含角色、任务、上下文、格式、约束的结构化 Prompt |
| Prompt优化 | 能定位输出问题并通过迭代、A/B 测试持续优化              |
| 工程意识   | 能理解系统 Prompt、结构化输出、版本管理与评测闭环       |
| 企业落地   | 能为真实业务场景设计可上线的 Prompt 方案                |

## 企业级学习原则

### 原则一：Prompt 不是“文案技巧”，而是产品能力设计

企业场景里，一个 Prompt 往往直接决定：

- AI 能不能稳定完成任务
- 输出能不能进入系统和工作流
- 风险是否可控
- 成本是否可持续

### 原则二：所有 Prompt 都要能被评测、复盘和迭代

| Prompt类型       | 不能只停留在   | 应沉淀为                     |
| ---------------- | -------------- | ---------------------------- |
| 基础Prompt       | 会写一句话指令 | 结构化模板                   |
| Role设计         | 会设定一个角色 | 系统人格与行为规范           |
| StructuredOutput | 会要 JSON      | 输出 schema 与校验规则       |
| Prompt优化       | 会改几句话     | 测试集、版本记录、评测结果   |
| 企业场景Prompt   | 会做 demo      | 上线策略、风险边界、兜底机制 |

### 原则三：Prompt 工程必须同时看效果、成本和风险

以下情况通常代表还没有达到企业级标准：

- 只追求效果，不看 Token 成本
- 只看单次 demo，不做稳定性测试
- 只写 Prompt，不定义输出格式和失败处理
- 只靠个人经验优化，不做版本管理和复盘

## 关键术语速查

| 术语                  | 说明                                             |
| --------------------- | ------------------------------------------------ |
| **Prompt**            | 提示词，用户输入给大模型的指令和上下文信息       |
| **System Prompt**     | 系统提示词，定义AI角色、行为约束和输出规范       |
| **User Prompt**       | 用户提示词，用户的具体问题或指令                 |
| **FewShot**           | 少样本提示，通过提供少量示例引导模型输出         |
| **ZeroShot**          | 零样本提示，不提供示例直接让模型完成任务         |
| **CoT**               | Chain-of-Thought，思维链，让模型逐步推理解决问题 |
| **Role Prompting**    | 角色提示，让模型扮演特定角色完成任务             |
| **Structured Output** | 结构化输出，让模型输出特定格式（如JSON）         |
| **Temperature**       | 温度参数，控制输出随机性（0确定性→1创造性）      |
| **Max Tokens**        | 最大Token数，限制输出长度                        |
| **Top P**             | 核采样参数，控制输出多样性                       |
| **Prompt Caching**    | 提示词缓存，复用已处理Prompt降低成本             |
| **Prompt Versioning** | 提示词版本管理，追踪Prompt迭代历史               |
