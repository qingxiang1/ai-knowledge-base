## 学习路径

### 按角色推荐

| 角色 | 必读章节 | 选读章节 | 学习目标 |
|------|---------|---------|---------|
| **AI产品经理** | 01→04→05→06→08 | 02、03、07 | 能设计系统Prompt、评估效果、管理版本 |
| **Prompt工程师** | 01→02→03→04→05→06→08 | 07 | 能设计高质量Prompt、优化效果、建立模板库 |
| **运营/业务人员** | 01→04→07 | 06、08 | 能使用模板快速生成内容、优化日常Prompt |
| **技术转型者** | 01→02→03→05→06→08 | 04、07 | 从技术视角理解Prompt工程、落地系统对接 |

### 按学习阶段

```
入门阶段（第1-2周）：
├── Prompt基础 → 理解Prompt组成要素和设计原则
├── Role设计 → 掌握系统Prompt设计方法
└── Prompt模板库 → 积累常用模板，快速上手

进阶阶段（第3-4周）：
├── FewShot → 掌握少样本提示，提升输出稳定性
├── ChainOfThought → 掌握思维链，提升推理能力
├── StructuredOutput → 掌握结构化输出，实现系统对接
└── Prompt优化技巧 → 建立持续优化方法论

实战阶段（第5-6周）：
└── 企业级Prompt案例 → 学习真实业务场景的Prompt设计
    └── 动手实践：为自己的业务场景设计完整Prompt系统
```

## 学习建议

1. **先学基础**：理解Prompt的基本构成和原理
2. **重点掌握**：Role设计和结构化输出是AI产品经理的核心技能
3. **掌握进阶**：FewShot和CoT是提升效果的关键技术
4. **积累模板**：建立个人/团队Prompt模板库
5. **实践验证**：每个技巧都要动手测试，记录效果
6. **持续迭代**：Prompt优化是持续过程，建立版本管理习惯
