## 学习建议

### 第一步：建立全局认知（1-2天）
1. **先读AI发展史**：了解AI从符号主义到大模型的演进脉络，建立时间线认知
2. **浏览术语表**：快速过一遍术语表，建立基本术语印象，后续遇到不懂的随时查阅

### 第二步：理解核心技术（3-5天）
1. **机器学习 → 深度学习**：从传统ML到DL的递进，理解"数据驱动"的范式转变
2. **NLP基础**：理解自然语言处理的核心任务，为大模型学习铺垫
3. **Transformer + Attention**：这是重中之重，理解了它就理解了大模型的"心脏"

### 第三步：掌握产品相关（2-3天）
1. **Token机制**：直接影响产品成本和交互设计，必须掌握
2. **Embedding原理**：RAG、搜索、推荐产品的技术基础
3. **LLM工作原理**：将前面所有知识串联，形成完整认知

### 不同背景的阅读顺序

| 你的背景 | 推荐阅读顺序 | 重点章节 |
|---------|------------|---------|
| 技术人员 | AI发展史(速览) → ML → DL → NLP → Transformer → Attention → Token → Embedding → LLM | LLM工作原理、Token机制（产品视角） |
| 传统产品 | AI发展史 → ML(精读) → DL → NLP → Transformer(精读) → Attention → Token(精读) → Embedding → LLM(精读) | Transformer、Token、LLM |
| 零基础 | AI发展史(精读) → ML(精读) → DL(精读) → NLP → Transformer(精读) → Attention → Token(精读) → Embedding → LLM(精读) → 术语表 | 全部（每章配合术语表） |

### 学习注意事项

1. **不要陷入数学推导**：产品经理需要理解"是什么"和"为什么"，而非"怎么算"
2. **结合产品场景理解**：每个技术概念都思考"这对产品设计有什么影响"
3. **术语表常备手边**：遇到不懂的术语随时查阅，不要跳过
4. **重点标记核心章节**：Transformer、Token机制、LLM工作原理是产品决策的高频知识
5. **对标能力模型**：学习过程中对照 [能力模型](../00-Roadmap/能力模型.md) 中「AI基础认知」维度进行自我评估
