## 学习建议

1. **按顺序学习**：建议从「产品经理认知」开始，逐步建立系统化认知
2. **结合实践**：每学完一章，尝试用所学方法分析一个实际产品
3. **交叉验证**：不同方法论之间可以相互印证，形成更全面的分析视角
4. **对标能力模型**：学习过程中对照 [能力模型](../00-Roadmap/能力模型.md) 进行自我评估，明确能力短板
5. **产出导向**：参考 [年度学习计划](../00-Roadmap/年度学习计划.md) 的产出清单，确保每个章节都有可交付的实践成果

| 角色 | 必读章节 | 选读章节 | 学习目标 |
|------|---------|---------|---------|
| **AI产品经理** | 产品经理认知 → 用户需求分析 → 用户画像 → MVP设计 → PRD编写规范 → 产品规划方法论 | 用户旅程图、JTBD理论、产品生命周期、产品案例分析 | 建立产品思维基础，掌握需求分析和产品设计方法 |
| **技术转型者** | 产品经理认知 → 用户需求分析 → MVP设计 → PRD编写规范 → 产品案例分析 | 用户画像、用户旅程图、JTBD理论、产品生命周期、产品规划方法论 | 从技术思维转向产品思维，掌握产品文档和验证方法 |
| **运营转型者** | 产品经理认知 → 用户需求分析 → 用户画像 → 用户旅程图 → JTBD理论 → MVP设计 | PRD编写规范、产品生命周期、产品规划方法论、产品案例分析 | 从运营视角转向产品视角，掌握用户洞察和产品规划 |

### 按学习阶段

```
入门阶段（第1-2周）：建立产品认知
├── 产品经理认知 → 理解产品经理的角色定位和核心价值
├── 用户需求分析 → 掌握需求挖掘和分析的基本方法
├── 用户画像 → 理解用户画像的构建和应用
└── 动手实践：为一个产品构建用户画像

进阶阶段（第3-4周）：掌握核心方法论
├── 用户旅程图 → 掌握用户体验分析工具
├── JTBD理论 → 理解用户真实动机和竞争分析
├── MVP设计 → 掌握最小可行产品的设计方法
├── PRD编写规范 → 掌握产品需求文档的标准写法
└── 动手实践：为一个功能模块编写PRD

实战阶段（第5-6周）：建立全局视角
├── 产品生命周期 → 理解产品各阶段的管理和决策
├── 产品规划方法论 → 掌握系统化的产品规划流程
├── 产品案例分析 → 从实战案例中学习方法应用
└── 动手实践：为一个产品做完整的产品规划
```

## 关键术语速查

| 术语 | 说明 |
|------|------|
| **产品经理** | 负责产品的规划、设计、开发、运营、迭代的全流程管理者 |
| **用户需求** | 用户在使用产品过程中期望解决的问题或满足的需求 |
| **用户画像** | 基于用户数据构建的典型用户模型，用于指导产品设计 |
| **用户旅程图** | 用户使用产品的完整体验路径，用于发现体验痛点 |
| **JTBD** | Jobs-to-be-Done，用户"雇用"产品完成任务的框架 |
| **MVP** | Minimum Viable Product，最小可行产品，用于快速验证产品假设 |
| **PRD** | Product Requirement Document，产品需求文档 |
| **产品生命周期** | 产品从诞生到消亡的全过程，分为引入期、成长期、成熟期、衰退期 |
| **产品规划** | 系统化的产品发展路线图，包含目标、策略、路径、里程碑 |
