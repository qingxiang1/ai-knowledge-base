# 产品基础

> 产品经理核心能力基础

本模块共 **10** 个章节，系统覆盖产品经理必备的基础认知与核心方法论。

---

## 本章定位

`01-产品经理基础` 是整个知识库的产品方法论底座，用来解决 3 个核心问题：

1. **知道产品经理如何定义问题**：避免一上来就讨论功能和方案
2. **知道如何把模糊需求转成可执行方案**：形成标准化分析、设计与文档能力
3. **知道 AI 产品与传统产品共通的基本功是什么**：保证后续学习 AI 能力时不“技术代替产品”

### 学完本章后应具备的最低能力

| 能力 | 达标表现 |
|------|---------|
| 问题定义 | 能区分用户表述、真实痛点与业务目标 |
| 需求分析 | 能完成基础访谈、整理需求并做优先级判断 |
| 产品设计 | 能设计 MVP，并说明为什么先做这些功能 |
| 文档表达 | 能编写一份结构完整、可进入评审的 PRD |
| 方法迁移 | 能把传统产品方法论迁移到 AI 场景中使用 |

## 学习建议

1. **按顺序学习**：建议从「产品经理认知」开始，逐步建立系统化认知
2. **结合实践**：每学完一章，尝试用所学方法分析一个实际产品
3. **交叉验证**：不同方法论之间可以相互印证，形成更全面的分析视角
4. **对标能力模型**：学习过程中对照 [能力模型](../00-Roadmap/能力模型.md) 进行自我评估，明确能力短板
5. **产出导向**：参考 [年度学习计划](../00-Roadmap/年度学习计划.md) 的产出清单，确保每个章节都有可交付的实践成果

## 企业级学习原则

### 原则一：先定义问题，再讨论方案

如果没有先把“用户是谁、业务目标是什么、当前问题是什么”说清楚，再好的功能设计也容易跑偏。

### 原则二：所有方法论都要能落到交付物

| 方法 | 不能只停留在 | 应沉淀为 |
|------|-------------|---------|
| 用户需求分析 | 看过访谈方法 | 访谈提纲、访谈记录、需求清单 |
| 用户画像 | 会画模板 | 真实场景下的角色画像 |
| 用户旅程图 | 知道概念 | 关键节点、痛点、机会点分析 |
| JTBD | 会背框架 | 可指导需求判断的任务陈述 |
| MVP设计 | 会说最小可行 | 功能取舍表、验证方案 |
| PRD编写 | 会用模板 | 可进入评审和研发沟通的方案文档 |

### 原则三：避免“假产品能力”

以下情况看似学了很多，实际上还不达标：

- 只会讲方法，不会做真实案例分析
- 会列需求，但不会做优先级取舍
- 会写功能清单，但不会写目标、范围、风险和验收标准
- 会分析用户，却不能把洞察转成设计动作

## 学习路径

### 按角色推荐

| 角色 | 必读章节 | 选读章节 | 学习目标 |
|------|---------|---------|---------|
| **AI产品经理** | 产品经理认知 → 用户需求分析 → 用户画像 → MVP设计 → PRD编写规范 → 产品规划方法论 | 用户旅程图、JTBD理论、产品生命周期、产品案例分析 | 建立产品思维基础，掌握需求分析和产品设计方法 |
| **技术转型者** | 产品经理认知 → 用户需求分析 → MVP设计 → PRD编写规范 → 产品案例分析 | 用户画像、用户旅程图、JTBD理论、产品生命周期、产品规划方法论 | 从技术思维转向产品思维，掌握产品文档和验证方法 |
| **运营转型者** | 产品经理认知 → 用户需求分析 → 用户画像 → 用户旅程图 → JTBD理论 → MVP设计 | PRD编写规范、产品生命周期、产品规划方法论、产品案例分析 | 从运营视角转向产品视角，掌握用户洞察和产品规划 |
| **零基础入门** | 产品经理认知(精读) → 用户需求分析(精读) → 用户画像 → MVP设计 → PRD编写规范(精读) | 其他章节 | 建立产品基础认知，掌握核心方法论 |

### 按学习阶段

```
入门阶段（第1-2周）：建立产品认知
├── 产品经理认知 → 理解产品经理的角色定位和核心价值
├── 用户需求分析 → 掌握需求挖掘和分析的基本方法
├── 用户画像 → 理解用户画像的构建和应用
└── 动手实践：为一个产品构建用户画像

进阶阶段（第3-4周）：掌握核心方法论
├── 用户旅程图 → 掌握用户体验分析工具
├── JTBD理论 → 理解用户真实动机和竞争分析
├── MVP设计 → 掌握最小可行产品的设计方法
├── PRD编写规范 → 掌握产品需求文档的标准写法
└── 动手实践：为一个功能模块编写PRD

实战阶段（第5-6周）：建立全局视角
├── 产品生命周期 → 理解产品各阶段的管理和决策
├── 产品规划方法论 → 掌握系统化的产品规划流程
├── 产品案例分析 → 从实战案例中学习方法应用
└── 动手实践：为一个产品做完整的产品规划
```

## 关键术语速查

| 术语 | 说明 |
|------|------|
| **产品经理** | 负责产品的规划、设计、开发、运营、迭代的全流程管理者 |
| **用户需求** | 用户在使用产品过程中期望解决的问题或满足的需求 |
| **用户画像** | 基于用户数据构建的典型用户模型，用于指导产品设计 |
| **用户旅程图** | 用户使用产品的完整体验路径，用于发现体验痛点 |
| **JTBD** | Jobs-to-be-Done，用户"雇用"产品完成任务的框架 |
| **MVP** | Minimum Viable Product，最小可行产品，用于快速验证产品假设 |
| **PRD** | Product Requirement Document，产品需求文档 |
| **产品生命周期** | 产品从诞生到消亡的全过程，分为引入期、成长期、成熟期、衰退期 |
| **产品规划** | 系统化的产品发展路线图，包含目标、策略、路径、里程碑 |
