# 月度学习计划

> 以月度为周期，将年度目标拆解为可执行的周计划，确保学习节奏稳定推进

---

## 一、计划说明

### 1.1 使用指南

本计划提供两种使用方式：

1. **跟随年度节奏**：按「年度学习计划」的季度主题，每月选择对应模块深入学习
2. **按需定制**：根据自身当前水平和目标，选择特定月份主题进行强化

### 1.2 月度学习节奏

```
Week 1：理论学习（输入）
    ↓
Week 2：实践操作（输出）
    ↓
Week 3：项目应用（整合）
    ↓
Week 4：复盘优化（提升）
```

### 1.3 时间投入建议

| 日期类型 | 建议时长    | 主要活动          |
| -------- | ----------- | ----------------- |
| 工作日   | 1.5-2小时   | 理论学习 + 小练习 |
| 周末     | 4-6小时     | 项目实践 + 复盘   |
| 月度总计 | 约60-80小时 | —                 |

---

### 1.4 月度执行核心原则

#### 1.4.1 原则一：每个月只解决一个主问题

不要把一个月拆成“什么都学一点”，而应明确本月最重要的主问题，例如：

- 我这个月要补的是 **AI 技术认知**
- 我这个月要补的是 **Prompt 设计与评测能力**
- 我这个月要补的是 **RAG 方案设计能力**
- 我这个月要补的是 **项目交付与作品沉淀能力**

#### 1.4.2 原则二：每周必须同时包含输入、输出、反馈

| 环节 | 典型动作                    | 目的           |
| ---- | --------------------------- | -------------- |
| 输入 | 阅读文档、课程、案例        | 建立认知       |
| 输出 | 写方案、做 Demo、做对比实验 | 形成能力       |
| 反馈 | 自测、他评、复盘            | 发现问题并纠偏 |

#### 1.4.3 原则三：月度计划必须能被验证

一个合格的月计划，至少要回答下面 4 个问题：

1. 本月结束时，我能拿出什么成果？
2. 这些成果如何判断是否合格？
3. 如果延期，我砍掉什么，保留什么？
4. 这些成果能否进入作品集或能力档案？

## 六、企业级执行标准

### 6.1 什么叫“这个月真的学到了”

以下状态分别对应不同成熟度：

| 状态   | 表现                             | 是否达标 |
| ------ | -------------------------------- | -------- |
| 只看过 | 读完了文档，但没有任何输出       | 否       |
| 能复述 | 能解释概念，但无法用于真实问题   | 基础达标 |
| 能应用 | 能在方案、Prompt、项目中正确使用 | 达标     |
| 能评估 | 能比较不同方案优劣并做取舍       | 良好     |
| 能复盘 | 能发现问题、总结经验、指导下一轮 | 优秀     |

### 6.2 月度成果的企业级最低标准

| 类型          | 低于标准的表现               | 达标表现                             |
| ------------- | ---------------------------- | ------------------------------------ |
| 需求/方案     | 只有功能罗列，没有目标与取舍 | 有目标、流程、指标、边界和风险       |
| Prompt资产    | 只有零散 Prompt              | 有分类、场景、输入输出示例、优化记录 |
| RAG/Agent方案 | 只有概念描述，没有流程细节   | 能讲清组件、数据流、评测与兜底       |
| 项目Demo      | 只能展示 happy path          | 能解释异常场景、限制条件和下一步优化 |
| 复盘文档      | 只有流水账总结               | 有结果、问题、根因、改进动作和负责人 |

### 6.3 延期时的处理原则

如果当月任务明显完成不了，优先级建议如下：

1. **保留核心成果**：如 PRD、方案、Demo、复盘
2. **压缩辅助输入**：如额外课程、扩展阅读、非关键案例
3. **不牺牲复盘**：即使延期，也要记录为什么延期
4. **把未完成项显式迁移到下月**：不要无声遗忘

## 七、常见问题

### 7.1 Q1：工作日太忙，如何保证学习时间？

**建议**：

- 将学习任务拆分为15分钟单元
- 利用通勤时间听AI播客
- 午休时间做概念卡片复习
- 周末集中完成大块实践任务

### 7.2 Q2：学到后面忘记前面怎么办？

**建议**：

- 每周五固定为复习日
- 使用Anki等记忆卡片工具
- 建立概念关联图（而非孤立记忆）
- 教别人是最好的复习方式

### 7.3 Q3：理论和实践如何平衡？

**建议**：

- 遵循「3:7原则」：30%时间学习理论，70%时间动手实践
- 每学一个新概念，立即找一个应用场景
- 用项目驱动学习，而非为了学习而学习

### 7.4 Q4：如何判断自己学懂了？

**自测标准**：

- [ ] 能用自己的话解释概念（不背定义）
- [ ] 能画出概念之间的关系图
- [ ] 能举出3个实际应用例子
- [ ] 能回答一个初学者的提问
