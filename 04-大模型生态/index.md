# 04-大模型生态

> 全面解析当前主流大模型家族的技术特点、产品生态与选型策略，帮助AI产品经理建立系统的模型认知体系

---

## 本章定位

`04-大模型生态` 解决的不是“知道几个模型名字”，而是以下 4 个更关键的问题：

1. **如何从业务目标、性能要求、成本预算与合规约束出发做企业级模型选型**
2. **如何理解闭源、开源、推理模型、多模态模型之间的真实差异，而不是只看排行榜**
3. **如何避免单模型依赖，把模型能力映射到具体产品架构、供应商策略和回退方案**
4. **如何把模型认知转化为产品方案、POC 评测、采购决策与长期能力建设**

## 企业级学习原则

1. **先理解能力边界，再讨论技术神话**：大模型不是越强越好，而是越适配越有价值
2. **先做候选池，再做 POC 验证**：企业选型必须通过真实业务数据验证
3. **先看总拥有成本，再看单次调用价格**：采购、接入、运维、风控和替换成本都要算
4. **始终保留替代路径**：供应商变化、价格调整、能力退化都要求有备选方案

## 阶段验收标准

- [ ] 能解释闭源、开源、推理模型、多模态模型的边界与适用场景
- [ ] 能基于真实业务场景筛选 3-5 个候选模型并给出理由
- [ ] 能设计模型选型 POC，包括样本集、指标、成本测算和风险评估
- [ ] 能输出一份企业级模型选型建议，包含主模型、备份模型和回退策略

## 最小作品集

建议学习完本模块后，至少沉淀以下 3 类产出：

1. **模型选型矩阵**：对 3-5 个候选模型按能力、成本、合规、生态和部署方式打分。
2. **POC 评测报告**：基于真实业务样本记录评测指标、Bad Case 和结论。
3. **供应商策略说明**：写清主模型、备用模型、切换条件和预算边界。

## 关键术语速查

| 术语                   | 说明                                                             |
| ---------------------- | ---------------------------------------------------------------- |
| **GPT**                | Generative Pre-trained Transformer，OpenAI的生成式预训练模型系列 |
| **Claude**             | Anthropic的大语言模型，强调安全对齐和长上下文                    |
| **Gemini**             | Google的多模态大模型，原生支持文本/图像/视频/音频                |
| **Llama**              | Meta的开源大模型系列，支持私有化部署和微调                       |
| **DeepSeek**           | DeepSeek公司的大模型，MoE架构，极致性价比                        |
| **Qwen**               | 阿里通义千问，中文能力强，全尺寸开源                             |
| **MoE**                | Mixture of Experts，混合专家架构，提升模型效率                   |
| **RLHF**               | Reinforcement Learning from Human Feedback，人类反馈强化学习     |
| **Context Length**     | 上下文长度，模型一次能处理的最大Token数                          |
| **Parameters**         | 模型参数量，衡量模型规模（如7B、70B、405B）                      |
| **API Pricing**        | API定价，按Token计费的价格策略                                   |
| **Private Deployment** | 私有化部署，在企业内部环境部署模型                               |
| **Fine-tuning**        | 微调，在特定任务数据上调整模型                                   |
| **POC**                | Proof of Concept，概念验证，测试模型在特定场景的效果             |
